# 在地金流四強比拚：綠界、藍新、TapPay、PAYUNi

## TL;DR

- **費率都擠在 2.8% 附近**——綠界 2.8875%（官網標 2.75% 未稅）、藍新 2.8%、PAYUNi 2.8%、TapPay 則走「年費 + 刷卡費」模式——表面差距不到 0.1%，真正拉開成本的是**提領費、年費、特約門檻**
- **訂閱制首選是藍新**：定期定額 + 卡號綁定 + 過期卡自動更新；綠界、TapPay、PAYUNi 也都支援定期定額，但自動更新卡號的可靠度與介接便利度，藍新仍是業界慣例
- **想要一頁式結帳、不跳轉**：只有 **TapPay** 把 iframe / Fields SDK 當成核心產品，其他三家仍以跳轉或 MPG（Merchant Page Generator）為主——這是 UX 上最大的分水嶺

## 費率與隱藏成本

截至 2026-04，四家公開費率如下：

| 項目 | 綠界 ECPay | 藍新 NewebPay | TapPay | PAYUNi |
|---|---|---|---|---|
| 信用卡（國內） | 2.8875%（含稅） | 2.8% | 年費制 + 刷卡費 | 2.8%（含稅） |
| 信用卡（國外） | 3.5% 起 | 3.5% | 同上 | 3.5%（含稅） |
| ATM 虛擬帳號 | 9-15 元/筆 | 類似級距 | 透過合作銀行 | 依級距 |
| 超商代碼 | 30 元/筆 | 類似級距 | — | 有支援 |
| 超商條碼 | 15 元/筆 | — | — | 有支援 |
| 提領手續費 | 15 元/筆（永豐帳戶 0 元） | 10 元/筆（每月前 5 次免費） | 視合作銀行 | 免設定費，撥款手續費依方案 |
| 年費 / 月費 | 免 | 免 | 約 NT$11,000/年未稅（業務個案可免設定費） | 免 |
| 新戶優惠 | 1.68-1.8% 限時 | 月營業額 >30 萬可談 2.3-2.5% | — | 首年手續費優惠方案 |

綠界的提領費 15 元看似小錢，但**訂閱制每月撥款幾百筆、每筆分開結算**時會累積可觀——這也是為什麼大多數綠界商家會直接開永豐帳戶，不是偏好，是成本優化。藍新每月前 5 次免費，對低頻提領的小團隊更友善；PAYUNi 則強調「天天撥款、發動當日到帳」，流動性最好，但詳細撥款手續費要看合約。

TapPay 走完全不同的路：**沒有 per-transaction 平台費，只有發卡機構刷卡費 + 年費**。這意味你的實際刷卡費可以談到 2.0-2.3%（要自己去跟收單銀行談），但要先把 NT$11,000 的年費吃下來。對月營業額 50 萬以上、能談到好費率的團隊來說，TapPay 長期成本最低；對月營業額 5 萬的個人開發者，反而最貴。

## 定期定額與訂閱扣款

SaaS 最在意的不是首刷費率，是**續訂成功率**。四家都支援信用卡定期定額，但能力有落差：

- **藍新**：支援卡號綁定（Token 化）、系統自動更新過期卡號（與發卡行串接）、失敗自動重試——這是訂閱制的事實標準
- **綠界**：支援定期定額，週/月/年期可設，但卡號更新仰賴消費者重新輸入比例較高
- **TapPay**：Card Token 機制設計完整，自動扣款 API 穩定，跨境多幣別是強項；但本地訂閱市場的生態支援（金流整合商、電子發票 plugin）不如綠界／藍新
- **PAYUNi**：支援定期定額與信用卡分期，官網有獨立 `/period` 產品頁，但第三方評測數據較少，建議小流量試跑

台灣消費者對訂閱制接受度約 15%（見首篇），這意味**每一次扣款失敗都是昂貴的流失**。若主力產品是月費 / 年費 SaaS，藍新的自動更新機制省下的客服成本，通常比省 0.2% 費率的其他家更划算。

## API DX 與電子發票整合

從開發者角度，四家的 SDK 成熟度差距比費率更大：

- **綠界**：官方 SDK 最齊（.NET / Java / PHP / Node.js / Python），社群套件也最多（Laravel、Rails 都有維護良好的 gem / package）；**電子發票整合是業界最完整的**——B2B、B2C、載具歸戶、48 小時內上傳財政部、ibon/FamiPort 列印，一站式
- **藍新**：官方 API 文件清楚，MPG（Merchant Page Generator）是多數人走的路徑——把結帳頁託管在藍新，串接最省事但 UX 得跳轉；電子發票是獨立服務、需另外簽約
- **TapPay**：開發體驗最「現代」，TapPay Fields 可做自訂 UI 的 PCI-DSS 合規表單，Direct Pay iframe 適合懶人；文件是英中雙語、GitHub 上有官方 example repo。**弱點是電子發票要串第三方**（多數團隊會搭配綠界發票或 ezPay 發票）
- **PAYUNi**：WooCommerce / 開店平台整合佳，但自建系統要多翻文件；電子發票可直接在同平台開通

> 實務經驗：若團隊是 Rails / Laravel / WordPress 背景、需要「卡 + 超商 + 發票」一次到位，綠界的 DX 摩擦最小；若是 React SPA / 行動 App 要做一頁式結帳，TapPay 幾乎是唯一解。

## 如何選：一句話決策

- **個人開發者、月營業額 < 10 萬**：綠界，免年費、生態最熟、Stack Overflow 答案最多
- **SaaS 訂閱制、月費 / 年費產品**：藍新，自動更新卡號省下的續訂率 > 任何費率優勢
- **行動 App / 一頁式結帳、跨境多幣別**：TapPay，年費吃得下就值得
- **電商 + 超取 + 7-ELEVEN 生態**：PAYUNi，統一集團自家整合最順，免設定費降低試錯成本

最後一個現實提醒：**官網標的都是牌價，月營業額 30 萬以上幾乎都可以談**。藍新、綠界的業務名片印出去，信用卡費率砍到 2.3-2.5% 是常態——先跑兩個月累積流量再去談，比開局就要優惠容易得多。
