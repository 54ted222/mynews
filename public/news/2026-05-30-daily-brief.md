[^statewright]: 5/27 在 Hacker News Show HN 衝上 120+ upvotes 的 AI agent 可靠性開源工具，由 Ben Cochran 主導。核心是 Rust 寫的 state machine 引擎、透過 MCP（Model Context Protocol）介接 Claude Code，把每個 state 能用的工具縮限到 5 個以內，並用前後 hook 強制提示與回饋。引擎與 agent layer 走 Apache 2.0，MCP gateway 採 FSL-1.1-ALv2 於 2029 年轉 Apache 2.0。

[^postgrest]: Supabase Data API 的後端，自動把 Postgres schema 投射成 RESTful endpoint，讓前端可以直接以 HTTP 對資料庫做 select / insert / update。權限完全依賴 Postgres 的 GRANT 與 row-level security——5/30 起新建 project 不再幫 public schema 自動發 GRANT，缺權限時 PostgREST 會回傳明確錯誤訊息並提示需要哪一條 GRANT 語句，而非靜默回 401。

[^workers-ai]: Cloudflare 的邊緣推論平台，把開源 LLM（Llama、Gemma、Kimi K 系列等）部署到全球 GPU 節點，開發者用統一的 REST / Workers binding 呼叫、按 token 計費。同帳號下可一鍵切換模型，但模型 alias 由 Cloudflare 控管，碰到 deprecation 會自動轉向新版——5/30 K2.5 自動 alias 到 K2.6 就是這個機制觸發的靜默 breaking change。

[^moe]: Mixture-of-Experts 的縮寫，一種把模型切成多個專家子網路、每次推論只啟用其中 2–8 個的架構。同等品質下 active 參數比 dense 模型少很多，所以 inference 成本與延遲都低，是邊緣推論友善的設計。Workers AI 免費 tier 主推的 Gemma 4 MoE 就是這個路線，常被當成 K2 系列的 fallback 選項。

[^nes]: Next Edit Suggestions 的縮寫，是 GitHub Copilot 在編輯器內預測「你下一步可能想改哪一行」的功能，行為類似強化版 inline completion。6/1 之後 Copilot 切到 AI Credits 計價，但 code completion 與 NES 被列為例外不扣 credit——亦即 IDE 內輸入時的補全與下一步建議仍維持原訂閱費包含、不會吃 token 預算。

[^win-ai-sdk]: 預期在 6/2 Microsoft Build 揭曉的 Windows on-device AI 整合套件，把 ONNX Runtime（跨硬體推論引擎）、DirectML（GPU 加速 API）、Copilot Runtime（Windows 內建 SLM 入口）打包成單一 NuGet 套件。目標是終結目前 Windows 上做 on-device AI「裝哪個 runtime、用哪個 API、走哪個硬體」的碎片化，讓 .NET / C++ 開發者一條相依就能跑離線推論。

[^mcp]: Model Context Protocol 的縮寫，由 Anthropic 2024 年底主導的開放協定，用 JSON-RPC 規範 LLM 與外部工具 / 資料源溝通的介面。任何 server 端實作 MCP 後，Claude Code 等支援 MCP 的 client 就能列出可呼叫的工具、檔案、prompt 範本。Statewright 透過 MCP gateway 介接 Claude Code，等於在協定層強制改寫工具清單。

[^foundry]: Microsoft 自家的 AI 開發平台（前身 Azure AI Studio），把 Azure OpenAI、開源模型、agent service、評估與監測整合到單一控制台。Agent Service 提供 hosted agent runtime、工具註冊、long-running 記憶等能力，並與 GitHub、Visual Studio、Windows on-device SLM 串接。Claude Dynamic Workflows 5/28 同步上線 Foundry，是 Anthropic 第一次把官方產品鋪進微軟自家 stack。

[^durable]: 一種 long-running workflow 的執行模型，把每一步驟的狀態與輸出寫進持久化儲存，process crash 或重啟時可從上次的 checkpoint 接續，不必整批重跑。代表實作有 Temporal、Inngest、Trigger.dev。Vercel Workflows 走「程式碼即工作流」路線，TypeScript / Python function 加裝飾子就能拿到自動重試、暫停續跑與觀測，無需另寫 DSL 或圖形編排。
