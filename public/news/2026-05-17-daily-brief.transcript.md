今天想聊的，是這幾天對 indie 開發者來說最該記住的五條線：OpenAI 把 ChatGPT、Codex、Developer API 合進同一個產品組，Salesforce 公開今年要燒三億美金在 Anthropic tokens，PwC 要訓三萬個員工拿 Claude 認證，然後 xAI 跟 Moonshot 同一週各端出一條新的 indie 棧——Grok Build CLI 跟 Kimi WebBridge，最後再加一條 ChatGPT 上的個人理財新功能。看起來題目散，其實全部都在告訴你同一件事：原本你站著的位置，這禮拜被往前擠了一格。

先講最大條的，OpenAI 五月十六號內部重組。President Greg Brockman 從代理改成永久接掌 product strategy，把 ChatGPT、Codex、Developer API 三條過去獨立的產品線合進同一個 product organization。Codex 之父 Thibault Sottiaux 升上來，接「unified core product across consumer, enterprise, developer surfaces」，等於一個人統整三條線。原本 ChatGPT 的負責人 Nick Turley，改去帶 enterprise products 跟 critical industries。Brockman 內部 memo 裡寫得很白，目標是「single agentic platform」、把 ChatGPT 跟 Codex 合成「unified agentic experience」，呼應今年三月華爾街日報報過的「desktop super app」——一個內建瀏覽器、能跑程式碼、又能對話、可以做多步任務的桌面 app，現在正式變成路線。

對 indie 來說意思很直接。過去半年靠「我幫你串 ChatGPT 加 Codex」這條 wrapper 賣點的人，產品藍圖一夜被蓋掉。但不要立刻悲觀，因為 OpenAI 自己的整合不是一夜完成的，Codex 會先吃 productivity tasks、再回頭吃 ChatGPT 那邊，所以中間還有六到十二個月的縫隙窗口，可以賣「Codex 加上你家 vertical workflow」這種比較專的東西。然後 UX 走向會更 power-user、更 CLI、更 dev-tool 化——你如果做的是非工程師 ChatGPT 使用者的介面與 onboarding，這條未來不確定性反而變高。

第二條是 Salesforce。CEO Marc Benioff 五月十六號上 All-In podcast，公開講今年 Salesforce 會花三億美金在 Anthropic tokens，幾乎全部燒在 coding。Salesforce 同時也是 Anthropic 投資人，投資金額另一個三億，大概一趴股權。重點不是這個數字本身——重點是這是第一次有 Fortune 50 等級的 CEO 把「我家 LLM 帳一年三億、幾乎全 coding」公開講出來。這給整個產業一個錨點：coding agent 的 token 帳會線性放大。然後 Benioff 同場呼籲建一個 intermediary layer router，把簡單任務分流給便宜模型、把 Claude 留給 hard reasoning。這句話一出，本來 model router 在過去六個月還算是節流選項、屬於少數人在意的事，這禮拜開始變顯學。OpenRouter、Portkey、LiteLLM、Bifrost、Morph 這幾家本來只有 infra 圈在用，接下來會被三十到五百人公司的 CTO 認真評估。

對 indie 來說有兩條可以接。第一條，幫客戶做三十天 router PoC audit，固定價五千美金，拿客戶過去三十天的 log、跑路由模擬、輸出一張策略表加上預估月帳省幅，然後再開六十天 build。第二條，Benioff 同時宣告 Slack 要內嵌 coding——這代表 Slack-as-IDE 變一條新接案題，靠 Slack 工作流的 ChatOps agency 可以開始準備 Slack-native 的 coding agent 範本。

第三條是 PwC。五月十四號 PwC 跟 Anthropic 宣布擴張結盟，要訓三萬個 PwC 員工拿 Claude certification、先美國再推全球三十六萬四千人；他們也設了一個 joint Center of Excellence，更重要的是推出 Office of the CFO，被定位成「PwC 第一個基於 Anthropic 技術的 standalone business unit」——意思是這條業務線不再附屬在 audit 或 advisory 之下，而是以 Claude 為核心獨立計價跟營運。公開的實證數字也很硬：保險核保的 underwriting cycle 從十週壓到十天，整體 delivery time 砍掉百分之七十。對 indie 的衝擊是這樣：你跑去跟 mid-market 客戶賣 Claude 整合，從本週開始 procurement 就會反問你「為什麼不直接讓 PwC 來做」。應對方法兩條，一是寫一頁差異化文件——我們比 PwC 快四倍、便宜十分之一；二是去打 PwC 沒進來的非美區市場，亞洲、日本、東南亞 mid-market 有大量空缺，直接寫一份「PwC 沒進來的五條空缺」內容稿。

接下來是兩條同週上的 indie 新棧。第一條是 xAI 五月十四號開的 Grok Build，是 terminal-native 的 agentic CLI，可以規劃、寫、跑、改、從自然語言把整個 app 蓋出來，最多八個 agent 並行；用 Grok 4.3 beta 當後端、上下文兩百萬 token、有 Plan Mode 可以先審後跑。定價走 SuperGrok Heavy 月費兩百九十九美金，前六個月引流價九十九。第二條是 Moonshot 五月十五號發的 Kimi WebBridge，是一個 Chrome 跟 Edge extension，把 LLM agent 接上你本機的瀏覽器——開頁、點按鈕、滾動、填表單、抽資料、截圖，全部資料不離本機，走的是 Chrome DevTools Protocol。它最特別的是 agent-agnostic，可以接 Claude Code、Cursor、Codex、Hermes、Kimi Code CLI 都行。同期 Kimi K2.6 在 SWE-Bench Pro 上拿了五十八點六分，贏過 GPT-5.4 的五十七點七跟 Claude Opus 4.6 的五十三點四。

這兩條為什麼一起講？因為它們同時在打開 Anthropic 跟 OpenAI 之外的備胎通道——一條是 coding CLI，一條是 local-first 的瀏覽器 agent。我建議跑 Claude Code、Cursor、Codex 為主棧的 indie，這禮拜開兩條 spike：一條用同 prompt 試 Grok Build CLI，比 plan-mode 體驗跟月帳；另一條把 Kimi WebBridge 裝起來，試一次「讓 agent 控本機瀏覽器抓已登入的 SaaS dashboard 資料」這種任務。特別是企業客戶常常擔心爬已登入 ERP 或金融帳號的合規問題，local-first 這條解法的價值會被放大。

最後一條是 OpenAI 五月十五號發的 ChatGPT Personal Finance，先給 Pro 用戶在美國 web 跟 iOS 預覽。它透過 Plaid 接了一萬兩千多家金融機構——Schwab、Fidelity、Chase、Robinhood、American Express、Capital One 都在內。dashboard 直接顯示 portfolio、spending、subscriptions、upcoming payments，你可以問「先還哪張卡」「未來金流規劃」「退休資產配置」。它目前是 read-only、不動錢。對美國做個人理財 wrapper 的 indie，這條基本上把這個 vertical 吃掉八成。但留下三條空檔：第一，非美區、亞洲歐洲 Plaid 沒覆蓋的市場；第二，主動代付，read-only 不動錢就代表訂閱取消、自動帳單操作這條沒人做；第三，跨幣別、加密資產、SMB 商業帳。這三條 sub-vertical 還能接續開。

把這幾條串起來看，重點是這樣：這禮拜 indie 開發者的處境，被三股力量同時擠了一下。一是大廠把產品線收成 super app——OpenAI 重組是訊號；二是 Big 4 拿 LLM 當 standalone 產品線——PwC 是訊號；三是大客戶把 token 帳公開——Salesforce 那三億是訊號。三條一起發生時，「我幫你串某兩個工具的 wrapper」這種定位的 indie 價值會最快被擠掉。要怎麼接？兩條路。一條是往大廠 6 到 12 個月還碰不到的縫隙鑽，比如 Codex 跟 ChatGPT 還沒完全合的那段時間、Personal Finance 還沒進的亞洲跟商業帳、PwC 還沒派人的非美 mid-market。另一條是吃這禮拜剛被推到顯學的新題目，特別是 LLM router PoC audit、Stripe Projects 上架代工、Codex mobile 的 vertical workflow 範本、Kimi WebBridge 的本機 browser agent 接案。哪條都比繼續守舊的 wrapper 賣點好。下禮拜五月十九、二十號是 Google I/O，會再有一輪訊號，我們再聊。
