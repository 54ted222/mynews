今天想聊 5 月 11 號到 5 月 14 號這四天，對亞洲 indie 來說最關鍵的一條主線。簡單說，就是「供應鏈簽章這條防線被打穿」開始，連著一串新訊號，最後落在「這週你最少要做哪 5 件事」。

先從 5 月 11 號 TanStack 那場攻擊講起。這次的編號叫 CVE-2026-45321，CVSS 9.6 分，等於是 critical 等級的最頂。攻擊發生在 UTC 晚上 7 點 20 到 7 點 26，短短 6 分鐘，攻擊者一口氣在 @tanstack 這個命名空間下，把 42 個 package 連著 84 個 artifact 全部換成惡意版本。攻擊團體叫 TeamPCP，手法分成三段：第一段是用 GitHub Actions 的 pull_request_target 觸發，這個觸發點本身就是長年被警告的危險面；第二段是把 Actions 的 cache 中毒，讓後續流程拉到動過手腳的內容；第三段最關鍵，他們從 runner 進程的記憶體裡，把 OIDC token 直接抽出來。

那為什麼這次特別嚴重？因為這是史上第一個帶著合法 SLSA provenance 的惡意 npm 套件。SLSA 是 OSSF 跟 Google 主導的供應鏈安全框架，搭配 sigstore 簽章，過去一直被當成「驗了就安全」的防線。@tanstack/react-router 每週下載量 1,270 萬，靠的就是這條防線。但這次攻擊者完整通過了簽章與 provenance 驗證，等於告訴所有人：光看簽章不夠。後面要補的是 release window，也就是 minimumReleaseAge 設成至少一天、加上 pnpm v11 預設禁掉 postinstall、加上主動審 lock file。

接下來在同一天，5 月 11 號，OpenAI 正式啟動了 The Deployment Company。這件事其實 5 月 8、10、11 連三天的 brief 都列在觀察區，5 月 11 號正式落地。初始投資 40 億美金、19 家合作夥伴，包含 Bain、Capgemini、McKinsey、Advent International 這些頂級顧問公司；OpenAI 自己持多數股權；最關鍵的是他們順手併購了一家叫 Tomoro 的公司，拿到第一批 Forward Deployed Engineer，也就是 FDE 軌道。這個動作要怎麼讀？要對照 5 月 4 號 Anthropic 跟 Goldman、Blackstone、Hellman & Friedman 合資的那家 JV，再加上 5 月 6 號 Anthropic 跟 EPAM 的合作。對比下來就清楚了：Anthropic 走「FDE 三條腿」——JV、EPAM、自家 Black Belt 計畫；OpenAI 走「FDE 一個獨立公司」。Mid-market 採購窗從這天開始被兩條軌道夾擊。對 indie 顧問來說，靠「我幫你導入 GPT」這種定位接案，會被 SI 體系直接壓縮。

再來，5 月 12 號 Android Show 上場，這裡端出三條對 indie 重要的軌道。第一條是 Googlebook，秋季出貨，Acer、Asus、Dell、HP、Lenovo 五家都做，預載的是 Gemini Intelligence。這個名詞是 Google 這次新公布的品牌稱呼，定位是「嵌入 OS 的 intelligence 層」，對標 Apple Intelligence。第二條叫 Create My Widget，今年夏天先在 Samsung Galaxy 跟 Google Pixel 上線。它是 vibe-code 介面，你用自然語言描述「我想要一個顯示今天 ToDo 的 widget」，Gemini 就直接生出來放到 Home Screen，這是 Android widget 史上第一次脫離 Java、Kotlin、KLWP 這些正式 SDK 路徑，由 LLM 即時生成 UI。第三條是 Gemini Omni，這是 Google 的影片生成模型，5 月 11 號在 Gemini app 內部的 UI string 跟 Pro tier 用戶帳號意外洩漏出來，預計 5 月 19、20 號 I/O 正式發。對 indie 的意義是，做 Android 短期內可以不必先寫 app，widget 變成一條新的 distribution 入口。

然後 5 月 14 號，Next.js 跟 React 出了 5 月的第三波 patch。這已經是三週內第三輪了——5 月 3 號第一輪、5 月 7 號第二輪、5 月 14 號第三輪。這次 13 條 advisory，涵蓋 DoS、middleware proxy bypass、SSRF、cache poisoning、XSS。攻擊面集中在 RSC，也就是 React Server Components 的反序列化，以及 App Router middleware。Next.js 推到 15.5.18 跟 16.2.6，React 對應到 19.0.6、19.1.7、19.2.6。對賣 Next.js boilerplate 或 SaaS template 的 indie 來說，這已經是「dependency pin 加 auto-PR 加 Playwright smoke test 標配化」的明確訊號。

同一天 5 月 14 號，Vercel 端出三條新訊。第一條是 AI Gateway 為 Claude Opus 4.7 開了 Fast mode，研究預覽階段，output token 生成速度大概快 2.5 倍，走 premium 計價、需要 waitlist、要帶 fast-mode-2026-02-01 這個 beta header。第二條是 Sandbox firewall 開始支援把特定 HTTP 請求 forward 到自管 proxy，還可以用 matcher 過濾哪些請求才走。對接企業 PoC 但要打客戶內網 API 的 indie，這條超實用。第三條是 Flags 推出 progressive rollouts，不再是固定 A/B 比例，可以排程「第 0 天 5%、第 3 天 25%、第 7 天 100%」自動跑。同一天 Anthropic 還公告了 Claude Platform on AWS，Messages、Files、Batches、Managed Agents、Agent Skills 全部接 AWS billing 跟 IAM；Claude Code 也補了 Bedrock service tier、claude project purge、ANTHROPIC_WORKSPACE_ID 這些企業向功能。

那這週你最少要做哪 5 件事？第一，掃自家 GitHub Actions 裡 pull_request_target 的用法，加上 persist-credentials: false，把 OIDC publish 步驟隔離成獨立 workflow，大概 45 分鐘。第二，列出你 lock file 裡所有 @tanstack 開頭的依賴，如果 5 月 11 號晚上 7 點 20 到 26 那個時間窗後安裝過，檢查 install 主機。第三，把 Next.js 升到 15.5.18 或 16.2.6、React 升到 19.x.6，跑一輪 Playwright smoke 再重新部署。第四，如果你重度互動式用 Opus 4.7，去排 Vercel AI Gateway Fast mode 的 waitlist，把適合切過去的 prompt loop 列出來。第五，如果你用 Vercel Sandbox 接企業客戶，挑一條打內網的 function，把 credentials brokering 切到 firewall forward 加 matcher，不要再直接放 env。

重點是，過去一週把「靠簽章 + provenance 就夠了」這個假設打掉，再用 OpenAI Deployment Company 跟 Anthropic JV 雙軌把 mid-market 採購窗洗一次，最後 Vercel 跟 Anthropic 同步把 Opus 4.7 Fast mode 跟 AWS 內生路線補齊。對亞洲 indie 來說，這週的工作不是追新模型，是把這 5 件最低限度的事做完，才不會在下一波 CVE 或下一個 SI 軌道上場時被動。
