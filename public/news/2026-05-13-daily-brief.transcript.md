今天想聊三條主線，這三條串起來看，會發現 enterprise AI 的玩法、前端供應鏈的攻擊面、以及消費端 AI 入口的版圖，剛好在 5/11 到 5/12 這 48 小時內各自前進一大步。

先講第一條，OpenAI 在 5/11 正式發射了一家叫 Deployment Company 的新公司。這件事我從上週就一直在追，原本大家在等的是 Anthropic 那邊跟 Goldman、Blackstone、H&F、Apollo、GIC、Sequoia 一起搞的那個 JV，公告日是 5/4，講好 5/14 之前要命名。結果 OpenAI 不演了，直接比 deadline 提前 3 天，一口氣端出三件套。第一件，新公司登記在 Delaware、估值 100 億美金、初始注資 40 億，找了 19 家投資人，TPG、Advent、Bain Capital、Brookfield 領投，OpenAI 自己 majority own。第二件，同一天宣布收購英國的應用 AI 顧問公司 Tomoro，一次拿到 150 名 forward-deployed engineer，這群人是真有產線經驗的，履歷上掛著 Tesco、Virgin Atlantic、Supercell。第三件最關鍵，旗艦客戶是 BBVA，西班牙第二大銀行，12 萬員工、跨 25 國，這不是 logo 牆上的擺飾，是 2025 年底就簽下 strategic partnership 然後全內部鋪 ChatGPT Enterprise、Code Interpreter、Voice、Operator 的真實案例，現在升級成 AI-native bank 的標竿。回頭看 Anthropic 那邊，到今天 5/13 還在用「AI-native enterprise services firm」這種通稱，5/14 還剩最後 24 小時，再不命名，這個 JV 大概就會走 IPO 揭露路線，變成不對外賣品牌、只走 PE 通路的東西。簡單說，enterprise AI 從紙上對撞變成實戰，FDE 這個派駐工程師的模式，從以前 Palantir 的標籤變成 Big Two 的預設姿態，mid-market 的 boutique 顧問跟單體 SaaS wrapper 的天花板，被狠狠壓低了。

再來講第二條，TanStack 的 npm 供應鏈大火。時間是 5/11 晚上 19 點 20 分到 19 點 26 分，UTC 時區，短短 6 分鐘內，攻擊者在 npm 上發了 84 個惡意版本，橫掃 42 個 @tanstack 開頭的 package，連 react-router 都中招，那個套件一週下載量大概 1,200 萬次。同一條攻擊鏈還打到 @squawk、@mistralai、@uipath，加起來 160 個以上的版本。重點是這次攻擊向量很新，不是去偷 npm token，而是先把 TanStack 的 router 倉庫 fork 成一個叫 zblgg/configuration 的名字，故意避開 fork list 搜尋，然後觸發 GitHub 一個叫 pull_request_target 的 workflow，這個觸發器很危險，它用 base repo 的 secrets 在執行、但如果不小心 checkout 了 PR 端的代碼，攻擊者的程式碼就會以高權限跑。攻擊者用這個切入點把 GitHub Actions 的 cache 中毒、等維護者後續合 PR 時還原中毒 cache，再從 runner 的進程記憶體，就是 /proc/<pid>/mem 那條路，把 OIDC token 抽出來，最後用合法的 OIDC token 走正常的 release pipeline 把那 84 個惡意版本推上 npm。所以即使有 signed provenance、防不了這招。安全研究團隊 StepSecurity 把這歸給一個叫 TeamPCP 的團體，這群人 3 月打過 Aqua Trivy、4 月打過 Bitwarden CLI，這次是 5/11 場。對你我來說最直接的影響是，如果你在那 6 分鐘前後有 CI build 或本機 install 過任何受影響版本，那不只 npm token、AWS、GCP、Kubernetes、Vault、GitHub、SSH 的所有 credential 都得輪換一遍，因為攻擊者拿的是 runner 上能摸到的所有 token。本週也建議把 GitHub Actions 裡所有 pull_request_target 的 workflow 全部 audit 一輪，能改成 pull_request 加明確權限的就改。

第三條主線，Google 在 5/12 開了 The Android Show: I/O Edition，端出三樣東西。最值得講的是 Googlebook，這是一條全新的 AI 筆電產品線，由 Acer、ASUS、Dell、HP、Lenovo 五家代工，預計 2026 秋季最快 9 月底上市，定位是 Gemini 原生的 AI 筆電，每台會配一個叫 Magic Pointer 的 AI 游標，鍵盤上還有一條叫 Glowbar 的標誌性光條，新 OS、跟 ChromeOS 共存。第二樣是 Gemini Intelligence，這是 Android 系統層級的智能層，定位類似 Apple Intelligence 跟 Microsoft 的 Copilot+，最有殺傷力的功能是 Chrome 內建的 auto-browse，你說一句要剪頭髮、它替你跨網站預訂完成。第三樣是 Android 17，重點是支援從 iOS 無線遷移 contacts、passwords、messages、apps、eSIM 過去，加上 Noto 3D emoji。傳聞中的 Aluminum OS 沒在這場端出來，應該是留給 5/19 跟 5/20 的 Google I/O 主 keynote。對中介型 SaaS 來說，Chrome auto-browse 是個立即衝擊，那些「我幫你比較、我幫你預訂」的入口會被瀏覽器本身吃掉，OTA、美容、票務、聚合訂閱這幾個領域要趕快衡量自家入口的暴露面。

其他幾條順便帶一下。Vercel 在這 48 小時改了 Sandbox firewall，加了 forwardURL 跟 vercel-sandbox-oidc-token 標頭，可以把 outbound HTTPS 轉到自家 proxy 並 audit，同時做 credential brokering，agent 程式碼根本拿不到 API key、由外層 inject 到 header，這對防 prompt injection 是個基礎建設層的解法。Vercel Flags 也加了 Progressive Rollouts、有 CLI 可以設定 10% 到 30% 到 60% 到 100% 自動推進。Stripe Sessions 那邊首波 ACP 跟 MPP 的 merchant 名單到位，包含 URBN 旗下的 Anthropologie、Free People、Urban Outfitters，還有 Etsy、Ashley Furniture、Coach、Kate Spade、Revolve、Halara、Abt Electronics，30 到 60 天內值得追 GMV 占比的揭露。

所以重點是，這 48 小時 OpenAI 把 enterprise 戰線從紙面拉到實戰、TanStack 事件把 npm 跟 GitHub Actions 的攻擊面整個翻新一層、Google 把消費端的 AI 入口直接植到瀏覽器跟筆電裡，三條線不同方向、但都在告訴我們 wrapper 跟中介層的日子愈來愈緊，該動的就這週動。
