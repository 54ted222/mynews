今天想聊三個 6 月 22 到 24 號之間發生、對 indie 開發者影響最直接的事件，加上一個必須訂正的烏龍。我盡量把表格跟數字用講的講清楚，你不用看畫面也能跟得上。

先講第一個重點，Cursor 在 6 月 22 號到 24 號之間把 3.9 版的 Customize 頁面 ship 出來。這件事為什麼重要呢？因為過去你用 Cursor 跑 vibecoding workflow，配置會散在很多地方，cursorrules 一個檔、commands 資料夾一堆 md、mcp.json 又一個、自己寫的 hook script 又是另一份，光是搞清楚哪個設定在哪裡就要花時間。3.9 之後，plugins、skills、MCPs、subagents、rules、commands、hooks 通通收進同一個 Customize 頁面，而且分成 user、team、workspace 三層管理。再來就更有意思了，marketplace 加了 leaderboard，也就是你可以看到自己團隊裡哪些 plugin 跟 skill 最熱門，一鍵就能 add；同時推一個叫 plugin canvas 的東西，簡單說就是可分享的「我們團隊 Cursor 標準配置模板」，這東西其實是個 PLG primitive，因為它可以病毒式傳播。第三件事，team marketplace 終於可以從 GitLab、Bitbucket、Azure DevOps 匯入，不再只能走 GitHub。對台日韓很多客戶內網不能用 GitHub 的場景，這代表你可以直接把客戶內網的 plugin repo 接進去，內網私部署 marketplace 的 audit 案子大概一千五到五千美金，加上月維運五百到一千五。如果你手上有教 cursor rules 的教學文，今天就值得把它包成 plugin canvas 上架收 PLG 流量，中文跟日文這塊現在供給接近零。

第二個重點是 Anthropic 的 Claude Tag 三件套，把「常駐 AI 隊友」這個棧第一次做到 production ready。6 月 23 號 Claude Tag 進 Slack，Team 跟 Enterprise plan 才能用、底模是 Opus 4.8。它不只是把 chatbot 釘在 sidebar，而是有自己的 service account、會保留團隊長期決策記憶、可以 ambient mode 自動 surface 資訊，也可以監看頻道、把任務非同步切段、跑完再回 thread。重點是舊的 Anthropic Slack app 8 月 3 號退役，從今天算起遷移窗剩 39 天，所以接下來七天到一個月，客戶遷移 audit 是熱 inbound。同梯還有兩件事撐起這個棧。第一件是 WIF，全名 Workload Identity Federation，6 月 17 號 GA。簡單說，過去你的 SaaS 要呼叫 Claude API 得拿一把長期 API key 塞 env var，這把 key 一旦洩漏就是永久災難。WIF 讓你直接拿 AWS IAM role、GCP service account、Azure managed identity、Kubernetes Service Account、或 GitHub Actions token 當身份，每次請求現換一張短期 token，洩漏面從「永久」改成「請求時短期」，整個風險量級不一樣。第二件是 Vercel Sandbox，6 月 24 號把單次最大運行時間從 5 小時拉到 24 小時，Pro 跟 Enterprise 才有。這代表 long-horizon 的 agent loop、end-to-end test pipeline、大型 batch processing 不必再自己切 chunk、不必自己寫 resume 邏輯。Claude Agent SDK 跟 Vercel AI SDK 直接吃到這個升級。把這三件套合起來——Claude Tag 在 Slack 當門面、WIF 處理身份、Sandbox 24h 跑長任務——對 indie 來說就是一個 audit 上架的好題目，固定價兩千到八千美金、加月維運五百到兩千。如果你自家 SaaS 還在用 static ANTHROPIC_API_KEY 環境變數，今天就該改 WIF。

第三個重點比較硬，是 6 月 23 號 Novee Security 公開揭露的 Cordyceps 漏洞家族。名字取自寄生真菌 Cordyceps，因為這類攻擊會「附身」在你的 CI/CD pipeline 上控制它。它不是單一 CVE，而是 GitHub Actions 跟同類 CI/CD 系統的一整類系統性可利用 pattern——命令注入、認證邏輯壞掉、artifact 投毒、跨 workflow 權限升級。掃了三萬個高影響 repo，找出三百多個完全可控的漏洞，Microsoft、Google、Apache、Cloudflare、PSF 都中標。攻擊鏈最毒的地方在於它多步、每一步單看都不像危險：用未信任的 PR 觸發一個低權限 workflow，輸出餵給高權限 workflow，最後拿到 cloud 全權 token。完全不需要登入，一個免費 GitHub 帳號就能 forge approval、push code、偷 credential。最致命的指控是：AI coding agent 大規模幫人生 CI/CD config，這些工具會重複複製不安全的 pattern，等於 AI 在橫向擴散漏洞。最危險的 trigger 是 pull_request_target，它跟一般 pull_request 不同，會在 base repo 的 context 跑、預設帶有讀寫 secret 的 token，如果你不小心 checkout PR 內容並執行其 script，就是把高權限交給未審查的程式碼。今天就該 audit 三類 trigger：pull_request_target、workflow_run、repository_dispatch，看它們跟低權限 workflow 之間的 artifact 跟 output 銜接有沒有破口。SAST 工具方面，社群已經有 Octoscan、zizmor、Poutine 三個 lint 工具，今天就該加進 pre-commit。對 dev tool agency 來說，「Cordyceps audit 加 AI-generated workflow 訂正」是這 7 天最熱的 inbound 題目，兩千到八千美金一單。

最後是訂正。6 月 23 號的 brief 寫「Fable 5 / Mythos 5 在 6 月 18 號透過 nationality-based access control 復原」，這跟事實不符。Anthropic 官方 6 月 24 號的 statement page 跟 explainx.ai 的分析都明確指出，Fable 5 跟 Mythos 5 仍然處於 6 月 12 號美國政府 export control directive 的 suspend 狀態，6 月 24 號全球所有使用者還是 offline。錯誤來源可能是把 Microsoft Azure Foundry availability docs 讀錯了。對 indie 影響有三件事要改：客戶 PoC 的 SLO 文件，四軌 fallback 把 Fable 5 整欄拿掉，改成三層——Opus 4.8 baseline，Sonnet 4.6 第二層，GLM-5.2 或 K2.7 Code 第三層，特定 vertical 補 GPT-5.5 或 5.5-Cyber；任何引用 Fable 5 規格的客戶提案今天就要重寫；「7 月 1 號 Anthropic 復活」的對賭過時，改看 export ban 何時解除，保守抓 8 月之前不要押。

重點是：Cursor 3.9 Customize 把擴展點統一一處、plugin canvas 是 PLG primitive、team marketplace 第一次支援非 GitHub；Claude Tag 加 WIF 加 Sandbox 24h 讓「常駐 AI 隊友」棧第一次 production ready、舊 Slack app 8 月 3 號退役遷移窗剩 39 天；Cordyceps 揭露 CI/CD 系統性漏洞、AI coding agent 是擴散源、今天就 audit pull_request_target 三類 trigger 加 Octoscan、zizmor 進 pre-commit；Fable 5 沒復原、SLO 文件四軌 fallback 拿掉那欄。這四件事這週都有時間窗口，動作快的人拿到 first-mover 流量。
