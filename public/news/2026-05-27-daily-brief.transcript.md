今天想聊三件對一人創業者跟小團隊工程師最有立即影響的事，全部都跟 AI agent 怎麼長出新的賺錢通路、還有一個逼在眼前的技術切換有關。先把這三條的脈絡講清楚，最後再收一個重點。

先講第一條，也是今天最重要的轉折。Google 的 Gemini Spark，就是它那個會在雲端二十四小時待命、能代你跑任務的 AI 助理，在五月二十五號確認了一件事：它去操作第三方應用程式的時候，走的是 MCP 這套協定。MCP 是 Anthropic 提出來的開放標準，簡單說，它就是讓 AI agent 跟外部工具、外部 app 之間有一套統一的接法，這樣同一個介接點就能被不同家的 agent 共用，不用為每一家各寫一套。

那 Spark 現在已經能接的有 Canva、OpenTable、還有 Instacart，等於你可以把 Google 文件裡的筆記直接丟給它生一份 Canva 簡報，或者依你收件匣的內容去訂個 OpenTable 餐廳，甚至幫你組一張 Instacart 採購清單，跨應用程式之間不用你再手動切來切去。然後它還宣布夏天會再補上 Adobe、三星、Spotify、CapCut、GitHub、Notion、Slack 這一票。

這件事有兩層訊號，都很關鍵。第一層是，Google 自己的旗艦級消費 agent 都採用了 Anthropic 的 MCP，這基本上就是替 MCP 蓋上「通用標準」的官方認證。你想想看，OpenAI、Google、微軟、Salesforce 在協定發布後十三個月內全部支援，公開的 MCP server 已經超過九千四百個，光是 SDK 一個月下載量就有九千七百萬次，七成八的企業 AI 團隊手上至少已經有一個跑在正式環境的 MCP agent。所以對 indie 來說，過去那種「我賭 MCP 會不會押錯協定」的擔憂，現在基本上可以不用煩惱了。

不過第二層訊號就比較刺一點。昨天，也就是五月二十六號那份簡報裡，我們講過一個論點，說 Spark 大概只接 Google 自家生態，所以像 Notion、Linear、或者你自己做的 SaaS 這種非 Google 的垂直領域，還是 indie 比較安全的利基。這個說法現在被部分推翻了，因為 Spark 夏天就要接 Notion、Slack、GitHub 了。

但壞消息反過來看其實是新機會。Spark 既然走 MCP，那意思就是：你只要把自己的 SaaS 包裝成一個 MCP server 暴露出去，就能被 Spark、被 ChatGPT 的 agent、被 Claude 這三家個人 agent 直接呼叫。換句話說，你的角色從「替每一家 agent 各寫一個連接器」，轉變成「做一個被三家共用的呼叫端點」。這是一條全新的散播通路，而且現在正是卡位的熱期，特別是那種窄垂直、在地化的，比如繁體中文、台灣法遵、特定產業流程的 MCP server，仍然是個好利基。

再來講第二條，是 Anthropic 的錢跟它的計畫。Anthropic 這個禮拜要收掉它的第二輪三百億美元募資，由 Sequoia、Dragoneer、Altimeter、Greenoaks 共同領投，估值衝到超過九千億美元。注意這是 2026 年內第二個三百億的輪次，二月才剛收第一輪、那時估值是三千八百億，等於十四週就翻倍了。它的年化營收跑率在四月七號就達到三百億，超過了 OpenAI 的兩百五十億。

不過對我們聽眾來說，比估值更該注意的，是它同一時間落地的幾個計畫。一個叫 Claude for Small Business，就是給小企業的套裝方案，內建連接器跟現成的工作流程，幫你把 Claude 塞進小公司日常用的工具。另一個是 Workday 基金會跟 Anthropic 合辦的個人創業者加速器，首批選十五個 solopreneur，給種子資金再加 Claude 額度。還有投入一億美元的 Claude 夥伴網路，以及一個大概十五億美元、找上 Blackstone、Goldman 這些華爾街巨頭的企業服務合資案。

這代表什麼？代表官方第一次明確地把賭注押在「一人創業、小型企業」這一層。對你來說有兩個具體的東西：一是真的有入口可以申請，去搶加速器或夥伴網路的資源；二是如果你在幫客戶做「為什麼要選 Claude」的決策表，現在可以再加一條——三百億到帳，而且官方親自下場做 solopreneur 跟 SMB 的通路。這裡也有在地化的空隙，因為這些計畫都是面向美國、英語市場的，台灣跟亞洲的一人創業者根本不知道怎麼申請，工作流程也沒有繁中版，這就是顧問可以切進去的地方。

最後第三條，是一個逼在眼前、最好今天就動手的技術切換。如果你在用 Cloudflare 的 Workers AI 跑 Kimi K2.5 這個模型，五月三十號,就是倒數三天，它會自動把 K2.5 的請求轉接到 K2.6。K2.6 是 Moonshot AI 出的新版開源模型，一兆參數、二十六萬多的脈絡長度，支援多輪工具呼叫跟視覺輸入。

重點是，這不是單純改個名字而已，它有真的會讓你程式壞掉的 API 變動。第一，控制思考模式的參數從原本的 enable_thinking，改成 thinking。第二，模型回傳的推理內容，從原本放在 reasoning_content 這個欄位，改成放在 reasoning 這個欄位。而且 K2.6 比 K2.5 貴。所以這三天之內，你要做三件事：把參數名改掉、把解析的欄位名改掉、然後重新估一下帳單會跳多少。如果不改，五月三十號那天它會靜默切換，你解析推理結果的程式可能就直接壞掉了。另外順帶一提，Supabase 五月也補了幾個更新，像是不接 Git 也能開分支變成預設、Agent Skills 開源、還有 Wrappers 升級；微軟的 Build 大會則是六月二號到三號，倒數六天。

重點是：MCP 已經成為跨廠牌的通用標準，你最該做的一件事，是把自己的 SaaS 暴露成一個 MCP server，去接 Spark、ChatGPT、Claude 這條新通路，窄垂直在地化的優先。Anthropic 三百億到帳的同時開了 solopreneur 跟小企業的官方入口，值得去申請也值得寫進你的客戶決策表。然後別忘了，五月三十號 K2.5 切 K2.6 只剩三天，今天就把參數跟欄位改一改、把帳單估一估，免得當天靜默壞掉。就這三件，動手吧。
