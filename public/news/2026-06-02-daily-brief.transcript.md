今天想聊 6 月 2 日這場 Microsoft Build keynote 落地之後，indie 開發者面前同時走起來的三座大時鐘——一座叫 Polaris swap，倒數九十一天；一座叫 Windows Agent Framework 改成 MIT 開源；還有一座是疊在前面兩天的計價地震上、今天又被 OpenAI Codex 砍半捅了一刀的「per-seat 真正死亡週」。我會把三條線從頭到尾講一遍，最後收尾講 indie 該怎麼動。

先講 Microsoft Build。今天上午九點半太平洋時間，Nadella 在舊金山 Fort Mason 開場，這場 keynote 真正的主角是 Project Polaris。簡單說，這是 Microsoft 第一次正式拿出自家研發的旗艦 coding 模型，架構走 MoE，也就是 Mixture of Experts，底下會依不同語言或框架切多個 sub-module，官方說在 HumanEval 跟 MBPP 這兩個 coding 基準上贏 GPT-4 Turbo，Rust 跟 Haskell 這種相對冷門但工程師重視的語言又特別強。重點來了：八月起，Polaris 會取代 GPT-4 Turbo 當 GitHub Copilot 的預設模型，原本的 GPT-4 Turbo 會保留三個月 fallback 期，所以從今天往八月底數，就是九十一天。這條時鐘對誰最痛？對所有靠「我家工具在 Copilot context 跑 GPT-4 Turbo」差異化的 indie 來說，九十一天之後預設模型一換，prompt 行為跟輸出 style 可能整個飄掉。而且這次 Polaris 定位很清楚——直接對打 Anthropic 的 Claude Code。要看 Microsoft 多認真，看內部就知道：他們從五月十五號開始取消五千個 Experiences and Devices 部門工程師的 Claude Code license，理由是每人每月燒 $500 到 $2000，把年度 AI 預算整個燒爆，六月三十號收尾日全部要轉去 Copilot CLI。八十四到九十五趴的內部採用率，就是 Polaris 的內部試煉。

第二件事是 Windows Agent Framework v1.0 改 MIT 授權開源。這套框架 Microsoft 四月已經放在 GitHub，但今天 Build 加掛 MIT、同步補 Python 跟 .NET 雙邊支援。最有意思的是它的抽象設計：agent 用 YAML manifest 定義，同一份描述檔可以從你的開發筆電起跑，escalate 到 Windows 365 的 GPU 節點，再往上 publish 到 Azure。Repo 是 github 上的 microsoft/agent-framework。配套的是另一個全新揭曉的 Azure Agent Mesh，這是一個 control plane，能夠跨本地 Windows server、Windows 365 Cloud PC、跟 Azure Arc edge 節點 federate，自動依延遲跟 GPU 可用度路由到最近節點，採 consumption 計價，2026 年 Q4 GA，現在進入半年預覽期。然後是 Copilot Workspace 退 beta 正式 GA，slash fleet 跟 autopilot 進 production，這代表多檔 PR 自動編修這條線終於可以給客戶用，不再是 research preview。比較弱的兩塊是 Windows Agent Runtime 跟 WSL 3 NPU passthrough：WAR 的 Insider preview 雖然六月就落地，但初期只支援文字 agent、輸入限 JSON、XML、PDF 三種，沒有 vision、也沒有通用 tool call，宣傳跟現況有落差；WSL 3 把 Linux kernel 放進輕量 VM 跑 NPU passthrough，但首批晶片只有 Qualcomm Snapdragon X Elite 跟 Intel 的 Meteor Lake、Lunar Lake，AMD 還要等。

第二條時鐘是 GitHub Copilot 的 AI Credits。昨天 6 月 1 號 LIVE，到今天剛好 24 小時，數字非常硬。官方討論串累計九百零四個 downvote、四百三十五則留言，正面票只有二十二。實案有三組數字最讓人坐不住：第一，跑 Sonnet 4.6 三小時就燒掉 35%、合計 2450 credits，跟前一天舊的 PRU 制比是 12 倍 burn rate；第二，Pro+ 用戶 19% 的月度上限數小時內消失，而前一個月整月用量也才 63%；第三，月帳單實案從 29 塊變 750、50 塊變 3000，最壞 50 倍。退訂跟切平台名單已經成形：OpenAI Codex、Anthropic Claude Code Max、DeepSeek V4 Pro、自託管 OSS、Cursor 的 Supermaven autocomplete 加 Composer 2.5，共識是「10 到 20 美金訂閱 unlimited frontier model」的時代 is gone, not coming back。更慘的是 Pro、Pro+、Student 三種方案從四月十七號起暫停新註冊，現有訂戶解約了就切不回去。

然後是今天疊上來的第三槍：OpenAI Codex Pro 的 2 倍 promo 昨天 5 月 31 號到期，6 月 1 號起同樣 100 美金一個月、capacity 砍半。原本「5 倍 Plus」的規格從今天起 effective 砍半，OpenAI 對 Enterprise 客戶丟兩個月 Codex 免費保大客戶，200 美金 Pro 仍是最高 usage 階。把這三條串起來——6 月 1 號 GitHub AI Credits、6 月 1 號 Codex Pro 砍半、6 月 8 號 Cursor Bugbot 切 usage-based——就是「per-seat 訂閱 unlimited」這個商業模式真正死亡的一週，三家旗艦同切。

最後是收容潮這一頭。Cursor Composer 2.5 是 5 月 18 號上線，輸入 0.5 美金、輸出 2.5 美金、fast variant 同智能但 3 跟 15 美金，比同級 frontier fast 便宜，宣稱用 25 倍合成任務量相對 Composer 2 訓練，sustained 多步比舊版穩，首週還在跑 double usage promo。配套是 Google 的 Chrome DevTools MCP public preview——這讓 AI coding agent 可以從 DevTools 介面直接看「程式跑起來在瀏覽器幹了什麼」加上 Performance Insights，Cursor、Claude Code、Codex 都吃得到。所以 Composer 2.5 在 6 月成了 Copilot 逃難潮最大的收容所。

重點是：對 indie 開發者來說，今天就要做三件事。第一，跑 GitHub Copilot 的，今天設 budget cap、列 7 天 burn rate dashboard，模型路由用 default Composer 2.5、Sonnet 4.6 跑 review、Opus 4.8 跑 migration，而且 Pro、Pro+、Student 千萬不要解約，解掉就鎖死。第二，跑 Codex Pro 100 塊一個月的，今天就算一下 5 月實際用量是不是超過砍半額度的一半，超過就切 200 塊的 Pro，沒超過就退 Plus 加掛 Composer 2.5 fast。第三，做 agent SaaS 中介的，今天 fork microsoft/agent-framework 試 YAML manifest 抽象，評估自家 SaaS 對齊 WAF 的可行性——這套抽象可以被 fork 套到 Bun、Deno、Linux container 等非 Windows 基建，雙向都吃。Polaris swap 九十一天倒數已經開始走，下個錨點是明天 6 月 3 號 Build Day 2 的 BRK207 議程，會深拆 fleet 加 autopilot，順便看 Microsoft 對 6/1 AI Credits 計價爆炸會不會丟補丁。
