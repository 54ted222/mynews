今天想聊兩條對小團隊、獨立開發者特別有感的事，一條是安全，一條是錢包。先講安全這條。

Anthropic 有一個叫 Claude Mythos 的模型，是他們今年四月揭露的，定位比 Opus 4.7 還要高一整階，是被刻意按住、不公開的受限前沿模型。它有多強呢？光是頭一個月，就在沒有人介入的情況下，自己找出了一萬個高風險或危險等級的漏洞，甚至還自主利用了 FreeBSD 一個藏了十七年的遠端執行漏洞。聽起來很科幻，但這就是它現在的能力。

過去這個模型只給大概五十個關鍵組織用，走的是一個叫 Project Glasswing 的封閉夥伴計畫，裡面有 AWS、蘋果、Google、微軟、摩根大通這種等級的成員，外面的人根本拿不到，價格也貴，是 Opus 4.7 的五倍。但五月二十七號出現了一個新訊號，好幾家媒體報導說，Claude Code 跟 Claude Security 裡面開始冒出 Mythos 的引用，這通常代表它可能在準備對外開放。同一天彭博社也發了一篇，標題大意是說新的 Claude 跟 Mythos 模型讓客戶感到不安，文章裡甚至引述了 Anthropic 執行長形容這模型「危險到不該發布」，員工也說它更像人、可能被拿去做網路攻擊。

那這對獨立開發者代表什麼？簡單說就是一把雙面刃。好的一面是，如果這種自主漏洞掃描的能力真的下放到一般人能用的 Claude Code 或 Claude Security，那「上線前幫你把整個程式碼倉庫掃乾淨、做安全加固」這種需求會暴增，這是個機會。但壞的一面是，攻擊者也會拿到同一套工具。所以那些沒有做過任何安全基線的一到三人小團隊，被自動化攻擊掃中的機率也會跟著升高。我的建議是，現在能做的事，就是趁 Mythos 還沒來之前，先用現有的 Opus 4.7 加上 Claude Security，把自己最關鍵的程式碼倉庫掃過一遍，當作一個基準線。如果你押注在 Claude 上，也記得在給客戶的簡報裡主動補上「模型治理、存取邊界」這塊，去回應大家的不安。

好，接下來講第二條，這條是真的會痛到錢包的。

時間鎖定在五月三十號，剩下兩天，Cloudflare 的 Workers AI 要把 Kimi K2.5 自動切換到新版的 K2.6。本來大家以為這只是改幾個 API 參數名稱的小事，像是把一個欄位從 enable_thinking 改成 thinking 之類的。結果真正的震撼是成本。K2.6 的計價是每百萬 token 輸入零點九五美元、輸出四美元。有一位開發者在社群發文，他跑的是一個上線中的正式系統，索引了十萬篇文件，結果官方只給了二十二天的通知，而且推薦的替代品還更貴。他原本那個「一個月只要五美元」的成本故事，跑一次批次處理就直接破滅了。

那他怎麼解決的？他把多文件的 reflection 處理層，整個搬到 Gemma 4 MoE 上面。這是 Google 開源的一個混合專家模型，重點是它在 Workers AI 上是原生支援、而且還留在免費的方案裡，所以不會被這次漲價波及。對那些靠 Workers AI 撐低成本批次處理或檢索任務的人來說，這是一條可以直接照抄的降本路線。

所以如果你正好在跑 K2.5，這兩天內一定要做決定，沒有模糊空間。要嘛今天就到測試環境去把參數名改好、重新估算帳單；要嘛就認真評估切換到 Gemma 4 MoE 或別的供應商。因為如果你什麼都不做，五月三十號的靜默切換可能會同時搞壞你的 reasoning 解析，帳單又跳升，兩頭燒。這件事也給了一個更大的教訓，就是「模型供應商抽象層加上成本告警」這種東西，已經從可有可無變成必需品了，別再把全部身家綁死在單一供應商的單一模型上。

除了這兩條主線，還有兩件事順帶提一下。一個是微軟的 Build 大會，六月二號到三號，剩五天，地點在舊金山。方向已經很明確了，Copilot 要重建成一個 agent 優先的多模型平台，會混搭 OpenAI、開源模型，還有 Windows 上的裝置端小型語言模型，也就是那種能在你電腦本地跑、主打低延遲跟隱私的輕量模型。做 Copilot 相關整合的人這週可以先備好觀察清單。另一個是 Anthropic 那筆三百億美元的第二輪融資，估值超過九千億，到五月二十八號早上都還沒看到正式宣布成交，所以那些等著申請小型企業通路的人，這週要繼續盯著。

重點是：這兩天先把 K2.5 的遷移決策做掉，別讓帳單偷襲你；同時趁 Mythos 還沒全面開放，先用現有工具把自家程式碼的安全基線建起來。攻防雙刃，防守的人動作要快一點。
