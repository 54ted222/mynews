今天想聊三件對 indie 創業者今天就會痛、不處理客戶這週就會撞牆的事。一個是 Fable 5 訂正、一個是 Noam Shazeer 從 Google 跳到 OpenAI、再來是 Cloudflare 一口氣放出來的三件套對台日韓 indie 真的第一次能跑 production。我們一條一條來。

先講 Fable 5 的訂正。昨天我們的 brief 寫的是「Fable 5 對所有 plan 仍維持 export-control suspended」，這個寫錯了，今天必須訂正。實際上 6/18 那天 Anthropic 已經透過一套組合拳把 Fable 5 跟它背後那個更高階的內部代號 Mythos 5 重新打開——條件是要做 nationality-based access control、API onboarding 要強制做身份驗證、還有強制資料保留。同一天 Anthropic 也在首爾開了亞太第三個辦公室，繼東京跟班加羅爾之後。然後從昨晚開始，6/22 晚上 11 點 59 分太平洋時間，那個給付費訂閱者的免費試用窗就收尾了，今天 6/23 起要用 Fable 5 就要燒付費 credit。

這件事對 indie 來說有三個立即的動作。第一個，如果你 6/9 到 6/14 之間升了 Pro、Max 或 Team 而且實際上跑過 Fable 5，今晚之前再跑一次你最硬的 agentic loop 留下 trace，明天起跑同樣的 prompt 比對 Opus 4.8 fallback 的觸發比例。為什麼要做這件事？因為 Tech Jacks Solutions 跟 The New Stack 報導開發者在 Reddit 跟論壇上回報，cybersecurity、chemistry、biology 這幾類 prompt 觸發 Opus 4.8 fallback 比 6/12 之前明顯變多，但 Anthropic 沒公布新的 fallback rate，所以你要自己量。第二個，如果你是賣台日韓 indie SaaS 給美系 enterprise 客戶的，這禮拜就寫一頁 FAQ 給銷售用，講清楚 nationality access control 會怎麼影響 PoC 流程，避免客戶 demo 當天 fallback 撞牆。第三個，寫一篇中文短文「Fable 5 為何 6/18 偷偷復原、台灣 indie 該怎麼測」丟 LinkedIn 跟 Threads，因為這題目中文供給目前是零。

第二條主線，Noam Shazeer 6/18 從 Google DeepMind 跳到 OpenAI，職位是 Lead for Architecture Research，講白了就是所有 OpenAI 模型底層神經網路結構的主導人。這個人有多重要？2017 年那篇 Attention Is All You Need 他是共同作者，Transformer 架構的奠基人之一。他之後共同創辦了 Character.AI 那家做虛擬角色聊天的公司，然後 2024 年 Google 用 $2.7B 一個 acquihire 把他從 Character.AI 撈回 DeepMind，現在 OpenAI 二度截胡。

對 indie 來說這代表兩個訊號。第一個，這通常是 OpenAI 架構層要大改的前置訊號，如果你的 LLM router 中長期權重押 OpenAI，這禮拜就把「2026 Q4 前可能要重排 GPT-5.x 系列權重」寫進客戶提案的 caveat。第二個更具體的，昨天 brief 提到 Polymarket 上「GPT-5.6 在 6/22 到 6/28 之間推出」的合約賠率從 83% 暴跌到 22%，那個就是即時的 sentiment 指標。再加上現在架構領導換手，OpenAI 短期出貨節奏不確定，那這禮拜你應該做的是把 LLM router 裡 Opus 4.8、GLM-5.2、K2.7 Code 這三軌的權重再壓上去，把 GPT 系列暫時降為「等 5.6 GA 上線後 24 小時重評」的狀態。

最後是今天最大的一塊，Cloudflare 6/18 跟 6/19 連著放出三件套，這是台日韓 indie 第一次有機會跑完整的 Cloudflare-only stack。第一件，6/18 PlanetScale 整合，PlanetScale 是源自 YouTube Vitess 團隊的 managed 資料庫，原本主打分片 MySQL、最近加了 Postgres。整合之後你可以在 Cloudflare dashboard 直接開 PlanetScale 的 DB，Worker 透過 Hyperdrive 連，帳單走 Cloudflare 帳，contract 用戶 7 月加入，原本 PlanetScale 的 dev branch、query insight、MCP server 都保留。第二件，6/19 Durable Objects 加了兩個 APAC 的新 location hint，一個叫 apac-ne 對應日本跟韓國，一個叫 apac-se 對應新加坡跟印尼。過去你用 apac 這個 default 等於「APAC 全分散」，單一 session 可能跨 RTT，現在可以把 DO 釘到 sub-region。第三件，同一天的 DO 連線保活，只要你 connect 或 outbound WebSocket 還 active，DO 就不會被 inactivity 驅逐，cap 是 15 分鐘。對 long-running 的 LLM agent workload 來說，這代表不會再 mid-stream eviction。

這三件套對 indie 有三個立即動作。第一個，如果你的 SaaS 已經用 Cloudflare Workers 加外部的 RDS 或 Supabase，這禮拜就試一下 PlanetScale 整合，估算把 Supabase、Vercel、Stripe 三家帳合一可以砍掉多少 admin overhead，通常會落在三成左右。第二個，如果你服務台日韓 user-base 的 agent SaaS，把 DO 的 location hint 加 apac-ne 或 apac-se 跑一輪，量一下 RTT 跟原本 apac default 差距多少。如果你跑出來像台北用戶從 apac 換 apac-ne RTT 從 380 毫秒降到 110 毫秒這種數字，那就是 case study 的素材。第三個，如果你在跑 long-running agent loop，把 streaming LLM call 從 fetch 改成 outbound WebSocket，這樣可以利用新的保活語意避免 60 秒以上 eviction 的問題。

今天值得做的有兩件事最關鍵，一個是升 Claude Code 到 v2.1.176 然後把客戶 PoC 的 SLO 文件統一改寫成 Fable 5 加 Opus 4.8 加 Sonnet 4.6 加 GLM-5.2 或 K2.7 Code 的四軌 fallback，第二個是 ship 一個「Fable 5 nationality control + Opus 4.8 fallback monitor」的 landing，月費 $49 或 $199，含 prompt classifier 跟 dashboard，順便開「客戶 PoC 重評 audit」$1.5K 到 $3K 的 inbound 表單。還有今天是歐盟 AI Act 高風險草案釐清版本公開意見徵詢的最後一天，CET 時間晚上 11 點 59 分截止，如果你賣 EU 客戶就花 20 分鐘送 5 行 feedback，重點問兩個問題，一個是高風險定義是否涵蓋 RAG over PII，另一個是 DPIA 範本能不能套用 micro-SaaS。

所以重點是三件事。Fable 5 不是還在 ban，是 6/18 已經偷偷復原但帶國籍限制跟更激進的 fallback，今天就要訂正提案。OpenAI 換了架構領導，加上 GPT-5.6 launch 時程崩了一半，router 權重要把 Opus 4.8 跟開源那兩軌壓上去。Cloudflare 一口氣補完 PlanetScale 整合、APAC sub-region DO、跟連線保活，台日韓 indie 第一次可以認真考慮 Cloudflare-only stack 撐 production AI SaaS。三條主線都帶 7 天熱期，中文供給為零，誰先 ship 就誰拿 inbound。
