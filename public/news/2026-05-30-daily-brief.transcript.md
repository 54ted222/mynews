今天想聊的，是一個對 indie 開發者來說特別緊的日子——5 月 30 號這一天，Cloudflare 跟 Supabase 同時各丟出一個 breaking change，而且是在同一天生效。先講第一個，Cloudflare Workers AI 上的 Kimi K2.5，從今天起會自動 alias 到 K2.6。聽起來像是版本升級沒什麼大不了，但其實有兩件事會咬到你。第一，價格漲了，新版本是輸入每百萬 token 零點九五美金、輸出每百萬 token 四美金，比舊版顯著漲，特別是輸出端漲到大概四倍。第二，參數名稱改了，原本叫 enable_thinking 的開關現在叫 thinking，原本叫 reasoning_content 的回傳欄位改成 reasoning。簡單說，你的程式沒改參數就會悄悄切到貴版本，而且解析 reasoning 那塊還會壞掉。所以如果你今天還在跑 K2.5，第一件事就是去 staging 改參數、跑一輪、看一週的帳單跳幅。

再來是同一天的 Supabase。從今天起新建的 project，預設不會再把 public schema 的 table 自動 grant 給匿名跟登入使用者了。換句話說，以前你建一個 table，前端 select 就拿得到資料，因為背後的 PostgREST 已經被授權；現在新 project 不會自動授權，前端打過去會直接被擋。好消息是 Supabase 這次做得很有禮貌，缺權限的時候 PostgREST 不會靜默回 401，而是回明確的錯誤訊息，告訴你缺哪個 role 的哪個 privilege、需要下哪一條 GRANT 語句。dashboard 上也多了一個叫做 Automatically expose new tables 的 checkbox，沒勾就是新預設、勾起來是維持舊行為當過渡。然後重點是，這個改動 10 月 30 號才會套用到既存 project，所以你還有大概一百五十天的緩衝去盤點舊 project 的 Data API 暴露面。對於跑 demo、跑 hackathon onboarding、或是有教學影片的人，這個影響很大——5 月 30 號之後新讀者照著舊腳本跑，會卡在「為什麼 SELECT 回 401」這一步，所以記得把明確 GRANT 那段補進腳本跟 starter。

接下來第二條主線，是 Claude Code 的 Dynamic Workflows。這東西 5 月 28 號剛揭曉，過了 48 小時，規格收斂得差不多了。最重要的兩個數字，一個是單次執行硬上限 1000 個 subagent，另一個是最多 16 個 concurrent，而且 concurrent 的數字會看你機器的 CPU 核心數往下降。第二個重點是它的工作機制——這不是「prompt 拆給多個代理人」，而是 Claude 會即時生成一段 JavaScript 編排腳本，然後丟給一個獨立的 runtime 在背景跑。這是一個全新的 primitive，是 LLM 第一次正式扮演「動態寫編排程式碼」的角色。它現在在 Max、Team、Enterprise 訂閱開研究預覽，同步上架 Anthropic 自家 API、Amazon Bedrock、Vertex AI、還有 Microsoft Foundry，三大雲都鋪齊了。

然後 Anthropic 給了一個很猛的旗艦案例——Bun 作者 Jarred Sumner 把整個 Bun 從 Zig 移植到 Rust。數字是這樣的：75 萬行 Rust 程式碼、從第一個 commit 到 merge 只花 11 天、原本的測試套件 99.8% 通過。做法是每一個 .rs 檔案配兩個 reviewer 平行跑。如果你是做 codebase migration 顧問的，這個案例就是你 deck 裡的硬通貨，可以拿去跟 SMB 或 mid-market 客戶談「我幫你跑一次百萬行 migration」這種服務。

第三條，6 月 1 號 GitHub Copilot 切換 AI Credits 計價，倒數 2 天。重點是這幾個。第一，退掉舊的 Premium Request 制，改成 token 計價，輸入、輸出、cached token 都算，每個模型按它的 API rate 換算成 credit，一個 credit 等於一美分。第二，code completion 跟 Next Edit Suggestions 這兩個 IDE 內的補全行為被列為例外，不扣 credit；但 Chat、CLI、Cloud Agent、Spaces、Spark 還有第三方 coding agent 都會吃 credit。第三，價格結構，Pro 維持 10 美金，Pro+ 39 美金含 39 credits，Business 19 美金含 19 credits，Enterprise 39 美金含 39 credits。其中 Business 跟 Enterprise 在 6 月到 9 月有過渡補貼，分別加碼 30 跟 70 credits。第四，既有的 Premium Request 預算會自動轉成 AI credits 預算，所以不用自己手動搬。對你來說今明兩天該做的，就是估算一下「6 月 1 號之後跑 7 天的 burn rate」，然後把 budget 上限鎖好，避免被意外的 Cloud Agent 跑爆預算。

再來是 Microsoft Build，6 月 2 號跟 3 號，地點在舊金山的 Fort Mason，倒數 3 到 4 天。今年 in-person 名額砍到 2500 人，比往年的三五千少一截，但 keynote 會免費直播。預期看點有三個：第一，GitHub Copilot 可能會揭曉所謂的 fleet mode 跟 autopilot，也就是 CLI 多任務自主跑、不需要 per-step 人手核可；第二，Windows 統一 AI SDK 預期把 ONNX Runtime、DirectML 跟 Copilot Runtime 包成單一 NuGet 套件，終結 on-device AI on Windows 的碎片化；第三，Azure AI Foundry Agent Service GA 之後的整合，預期會有 Foundry 配 Claude Dynamic Workflows 的 demo。Nadella 6 月 2 號 10 點 PT 開幕。如果你做隱私、離線、低延遲的垂直，像是診所、事務所、製造業，這個 Windows AI SDK 的方向值得 24 小時內就跑「on-device SLM 一小時上手」的內容搶時效。

總結一下這週的節奏，重點是這樣：今天 5 月 30 號是「同一天踩兩個靜默 breaking」的高峰；明天 5 月 31 號 GH-600 cert beta 截止；6 月 1 號 GitHub Copilot 換計價；6 月 2、3 號 Microsoft Build；五天內五件事疊一起，這週的 indie 等於做一場壓力測試。如果你今天只能做一件事，就是去把 Workers AI 的參數改好、把 Supabase 新 project 的 GRANT SOP 寫起來，這兩件事直接決定你接下來一週有沒有「為什麼壞掉」的 horror story。其他的，等 6 月 1 號跟 6 月 2 號到了再對應處理。
