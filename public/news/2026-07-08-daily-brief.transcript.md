今天想聊三條看起來各自獨立、但湊在一起會直接改變台灣獨立開發者接下來一週怎麼賣客戶的線。第一條是 Anthropic 的身分驗證，也就是 KYC，今天七月八號正式在 Claude 的消費 tier 上線；第二條是 Claude Fable 5 走 credits 制的第一天，一個大家昨天沒注意到、但今天被多方新聞交叉確認的「pool 一空就死機」陷阱；第三條，跟 AI 沒直接關係，但對台灣做服務業 SaaS 的人非常有感，就是政府補助 SIIR 五月二十號已經把整年度經費用完、明文不收第二梯次，讓中小企業 AI 補助能拿的組合縮成三軌。工具動態的部分留到最後幾句話帶過。

先講 Anthropic KYC。今天七月八號起，Claude 的 Free、Pro、Max 這三個消費 tier 帳號會開始被要求做身分驗證。不過有三個細節是這一波最容易被誤讀的。第一，這次 scope 只限消費 tier，Team、Enterprise、Platform 是走 Commercial Terms、直接豁免。第二，這才是最關鍵的一條，Claude API 完全豁免，因為 API 是用 API key 驗證、不進消費 tier 的帳號系統，所以你只要客戶架構是「B2B 前端接消費 tier、後端呼叫 API」，那你今天完全零 flow 影響。反過來說，只有那種「消費 tier 直接連給終端使用者用」的 SaaS，才會吃到最後一波的 flow breakage。第三，觸發不是全體強制、是先挑一小批被 flagged 的帳號、而且提供 appeal 路徑。生物特徵的部分由 Persona 服務端保管，Anthropic 目前沒公布保留時間軸。

那對台灣獨立開發者有什麼 immediate 的意義？我覺得有三條可以馬上動。第一，今天是黃金二十四小時 first-mover 窗，中文跟日文的「KYC Day 0 實測 + API 完全豁免遷移路徑」case study 供給幾乎為零，你今天寫、明天發，就是搶最前面。第二，你可以立刻對客戶開一條顧問線，叫做「Claude 消費 tier 轉 API 遷移 audit」，順便配 Claude Apps Gateway 這個 self-hosted 選型，一次收五百到兩千美金、再賣客戶 SOP 九十九到一百九十九塊 lifetime。第三個最急，如果你的客戶有超過三成的用戶是在中國、又拿不出護照，那今天起就要準備把 core flow 從 consumer Claude 拔掉，改走 Bedrock 或 GCP 加 API key 這條線，因為中國國家身分證不被 Persona 接受，這些人今天起就是被鎖在門外。

再來，Fable 5 的 credits 制。昨天七月七號正式從 Pro、Max、Team 的週限保護窗切成 credits pool，價格是每一百萬 token 十美元 input、五十美元 output，換算下來是 Opus 4.8 的兩倍、Sonnet 5 促銷價的五倍，是 Anthropic 現在最貴的 GA 模型。這件事本身不新，但今天七月八號有一個新細節落地，而且是 Techtimes、Apidog、Fable5.app 三個來源交叉確認的，就是：如果你沒有預先啟用 usage credits，那 Fable 5 access 會在 pool 燒完的當下、mid-session、直接停掉——沒有自動退到 metered billing 的 fallback。簡單說，就是你的 agent loop 半途死機。這個影響非常直接：重度依賴 Fable 5 跑大 context、跑長 agent loop 的 indie，如果 credits 沒 pre-fund，那你這個月的月費從七月八號起會 5 到 10 倍化，或者你 agent 就是在客戶面前掛掉。

Reddit 早期用戶的怒火也很明顯，原本 Anthropic 承諾兩週免費試用，結果六月十二號因為出口管制中斷，最後只剩七天、還是半週限的用量，有一則被大量引用的抱怨是「我們只用了三天、卻被承諾了十四天，現在剩七天還砍半」。所以對台灣 indie 有三個動作：一個是加一條新的 placeholder 進客戶 SLO，叫 `FABLE5_CREDITS_HARD_STOP` 等於 TRUE；第二是 ship 一個「Fable 5 credits pre-fund 檢查 + Sonnet 5 default 切換 SOP + agent loop mid-session hard-stop 檢測」的顧問包，五百到一千五百美金；第三，一樣是二十四小時 first-mover，中文日文的 case study 現在供給是零。而且 Sonnet 5 對 Fable 5 便宜五倍，是最直接的 fallback，你順便可以幫客戶把 router config 切一鍵化。

第三條線是台灣 SIIR。SIIR 就是服務業創新研發計畫，一般案最高 150 萬、聯合案 500 萬、國際合作 1000 萬，是台灣中小企業補助 stack 裡最大的一條線。但是 115 年度的補助經費，已經在五月二十號用完了，經濟部明文不受理第二梯次。也就是說，六月起 SIIR 這條實質上就是關的，但市面上還有很多顧問跟 SaaS 還在寫「SIIR 為主 pipeline」的 narrative，這批人 pipeline 已經斷了、只是還沒意識到。同期還在收案的中小企業 AI 補助 stack 收斂成三軌 default：一，商業發展署的服務業 AI 升級方案，單店十萬、連鎖三百萬、三處場域整合最高兩千萬，持續開放；二，商業發展署 30 人以下服務業數位轉型培力補助，這條是十二小時課程加可搭購軟體，2025 年 12 月 31 號截止申請，2026 年 3 月 31 號前要完成培訓；三，中小企業處的雲市集跟多元振興，最高十萬台幣導入 AI 工具，持續開放。

所以對台灣 indie，一樣是三個 immediate：第一，七十二小時 first-mover 寫一篇「SIIR 五月二十號用罄 → 中小企業 AI 補助 stack 三軌替代表」的 case play，供給也是零，narrative 點就是「SIIR 六月起實質關閉、TAI1 七月十號倒數兩天、商業署三軌成為 default」；第二，如果你已經在賣服務業 SaaS，那「十萬預算補助配套包 + 30 人以下培力補助講師方案」對你是最短鏈；第三，如果你在做政府 AI 補助配對 SaaS，那「補助案 real-time 開關狀態表」現在是最強的差異化功能，SIIR 五月二十號關案，就是最漂亮的示範信號熱期。

工具動態的部分快速帶過。Cursor 官方 changelog 補了 Auto-review 進新用戶 default，agent 產生的 diff 會自動 self-review，既有用戶要去 Settings → Agents 手動開。Team marketplaces 擴到支援 Team MCPs 加 organization group，admin 設定一次同步到 cloud agents、agents window、IDE、CLI。Claude Code v2.1.202 七月六號 release 新增 Dynamic workflow size 進 /config，加上 workflow.run_id 跟 workflow.name 兩個 OpenTelemetry attribute 進 workflow-spawned agents 的 telemetry，意思是一次 workflow run 全部 agent 的活動可以從 OTel 資料重建，Grafana、Honeycomb 直接對接。Product Hunt 有三個新品可以留意：TryCase 是 AI agent 專用的 disposable 測試環境，Retrace 讓 agent run 可以 replay 加 fork，Stigg 2.0 是 AI 產品 usage metering runtime。

重點是：今天七月八號有三個時間敏感窗要抓。一，Anthropic KYC 生效當天，「Claude API 完全豁免」的 narrative 是這一波最大的誤讀空隙，抓住二十四小時 first-mover。二，Fable 5 credits Day 1 的 hard-stop 陷阱是新細節，客戶那邊沒人在講，你今天檢查客戶有沒有 pre-fund credits 就是硬需求。三，台灣 SIIR 五月二十號關案這件事還沒被中小企業界完全 internalize，你這週寫「三軌替代表」就是最漂亮的信號套利。三條線加起來，是接下來七天你可以最快變現的三個角度。
