[^fennec]: **Fennec** 是 Anthropic 內部給 Claude Sonnet 5 的 codename。Anthropic 以動物為各模型家族命名（如 Opus、Sonnet、Haiku 對應內部代號），Fennec 是 6/30 對外發布的 Sonnet 5 在開發階段的代號；外部市場名稱仍為 Claude Sonnet 5，model id 為 `claude-sonnet-5`。

[^admt]: **ADMT（Automated Decision-Making Technology）** 是 Colorado SB 26-189（2026/5/14 簽署）採用的新監管框架，取代原 SB 24-205 的「high-risk AI system」分類。ADMT 走 disclosure-and-rights 路線——要求業者揭露使用自動化決策技術的事實、給予消費者解釋與申訴權利，而非 EU AI Act 式的事前風險分類與合規義務。2027/1/1 才正式生效。

[^swebench]: **SWE-bench Verified** 是 OpenAI 維護的 SWE-bench 子集（500 題），從原 SWE-bench 2294 題中由人工挑選、確認題目描述清晰且測試案例可靠的真實 GitHub issue。被視為衡量 LLM 軟體工程能力最常被引用的 benchmark；Sonnet 5 取得 82.1%、Sonnet 4.6 為 77.2%。

[^osworld]: **OSWorld-Verified** 是電腦使用（computer use / desktop automation）benchmark，測 agent 在真實 Ubuntu / Windows / macOS 桌面環境中完成跨應用程式任務的能力（開檔、瀏覽器、Office、終端機等）。人類專家 baseline 為 72.4%；Sonnet 5 取得 88.3%，是 desktop automation 第一次大幅超越人類。

[^acp]: **ACP（Agent Client Protocol）** 是 Cognition 主導的開放協定，定義 IDE / 編輯器與 local coding agent 之間的溝通介面——讓 Devin Local 等 agent 不綁定特定編輯器、第三方 IDE 也能整合相同 agent backend。對標 Anthropic 的 MCP（agent 與外部資料／工具）但聚焦 IDE-agent 連線。

[^sep1865]: **SEP-1865** 是 MCP（Model Context Protocol）規格提案編號，定義 MCP Apps——讓 MCP server 可回傳 sandboxed iframe UI 元件給支援的 client（如 Claude、Cursor）內嵌呈現，把 server 端能力升級為「server + 內嵌互動 UI」。被納入 MCP 2026-07-28 RC，Vercel AI SDK 7 已預先整合。
