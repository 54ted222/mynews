[^wif]: **WIF（Workload Identity Federation）**：雲端身份聯邦機制，讓某個運算工作（VM、容器、CI runner、Kubernetes Pod）用自身已有的雲端 / OIDC 身份去向另一個服務（例如 Claude API）換取短期、有作用域限制的存取憑證，省掉「複製貼上長期 API key」這個高風險環節。Google Cloud、AWS STS 多年前已有類似機制，Anthropic 在 2026-06-17 把它做進平台。

[^oidc]: **OIDC（OpenID Connect）**：建立在 OAuth 2.0 之上的身份認證協議，由 identity provider（如 Okta、Google、GitHub Actions）簽發 JWT 格式的 ID token 證明「這個請求方是誰」。WIF 多半以 OIDC token 為信任根：對方驗 token 來源與簽章後，就願意核發自家服務的短期憑證。

[^cordyceps]: **Cordyceps**：Novee Security 於 2026-06-23 公開的 CI/CD 系統性漏洞家族代號，名稱借自能控制宿主行為的寄生真菌。它不是單一 CVE，而是一類「命令注入 + 跨 workflow 權限升級 + artifact 投毒」可組合的攻擊 pattern，常見於 GitHub Actions 等自動化系統。

[^prtarget]: **`pull_request_target`**：GitHub Actions 的 workflow trigger，與一般 `pull_request` 不同之處在於它會在 base repo 的 context 執行、預設帶有讀寫權限的 secret 與 token。設計上是為了讓外部 fork PR 也能拿到必要憑證跑 label / comment，但若不小心 checkout PR 內容並執行其 script，就會把高權限交給未審查的程式碼，是 Cordyceps 攻擊鏈最常見的入口。

[^polymarket]: **Polymarket**：以加密貨幣結算的預測市場平台，使用者用 USDC 對未來事件下注（選舉、產品發布、模型 GA 日期等），合約價格被市場視為事件機率的反映。AI 圈常拿它的賠率當「市場共識預期」的參考訊號，但流動性與群體偏誤都需打折看待。
