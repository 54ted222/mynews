# 定價模型：訂閱率 15% 的台灣，怎麼設計收得到錢的價格

把產品蓋好、合規牆過了、整合層接了、客戶找到了——剩最後一道題：**訂多少錢、按什麼週期收**。台灣本地 SaaS 的定價不是國際模板的延伸，是另一套生態：消費者對訂閱接受度低、中小企業偏好年費或買斷、信用卡定期定額退訂率高、統一發票會把月付的帳搞得複雜。這篇對照 iCHEF、Ragic、漸強實驗室的實際定價，理出 indie 在台灣可以收到錢的訂閱模型。

## TL;DR

- **台灣消費者訂閱接受度估約 15%**（參照在地 SaaS 公開的 retention[^retention] 與多家市調共識），遠低於北美 40%+ 的水準。中小企業老闆更明顯偏好「**年費一次付清**」或「**一次買斷**」，月費對他們是「會被忘記但持續扣款」的麻煩
- **定價模型由你的客戶結構決定**：B2C 走「**月付 + 年付折扣 + lifetime deal**」三層；B2B SMB 走「**年費為主、月費為輔、人頭計價**」；垂直 niche（餐飲、診所、製造）走「**Setup Fee + 年費 + 加值模組**」這種更接近 ERP 的結構
- **統一發票把月付變複雜**：每月開一張發票對小公司行政成本顯著、對客戶報帳是噪音。年費年初一次開、月費走金流平台代開、AI usage-based 走儲值制——這些行政選擇與定價是綁在一起的

## 為什麼台灣消費者「訂閱厭惡」更明顯

台灣消費者不是不訂閱（Netflix、YouTube Premium、Apple Music 在台滲透率不低），是**對在地產品的訂閱接受度顯著更低**。原因可以拆三條：

第一，**信用卡定期定額心態**。台灣信用卡持卡率高，但對「**綁定每月扣款**」存在強烈警戒——因為早年電信、健身房、雜誌訂閱的定期定額爭議多，「訂了取消困難」是普遍記憶。對新出現的 SaaS 產品，使用者第一次按下「訂閱」需要的信任成本，比北美高得多。

第二，**小公司的成本邏輯**。中小企業老闆做財務決策的習慣是「**年初編預算、年末結算**」，月費 SaaS 在報表上會散落 12 行，年費則是一次性費用，相對好處理。再加上記帳士／會計師對年費的處理熟悉，月費反而會被退回去問「為什麼這筆每月都有」。

第三，**買斷情結**。資訊系統是台灣中小企業歷史上習慣的「**買回來放著**」資產（早年的 ERP、POS 都是賣斷）。「我付了你的錢但不擁有它」這個概念，對 50 歲以上的老闆需要解釋很多次。Ragic[^ragic] 在 SMB 市場走得通，部分原因就是它的價格結構（**用戶數計價 + 永久保有自己的資料 + 隨時可關 / 可降級**）對買斷情結的緩解。

> 訂閱接受度低不代表台灣不能做 SaaS，**代表你需要設計一個讓使用者「感覺像在買一個東西」的訂閱**。

## 三種台灣 case 拆解：iCHEF、Ragic、漸強實驗室

不同產品在不同客群面前發明了不同的定價結構。三個本土 SaaS 是現成的對照組：

**iCHEF（餐飲 SaaS）**：以雲端 POS 為核心、月費結構（依方案約 NT$1,000–4,000 / 月），加上**硬體 + 安裝設定費**作為 Setup。它在 SMB 餐飲老闆中能跑起來的關鍵不是月費低，是**「這台 POS 機」是看得見摸得到的物理感**——老闆覺得自己買的是「機器 + 軟體」，月費是維修費，這個心理錨點讓訂閱接受度大幅提高。截至 2026 年，iCHEF 服務 15,000 家餐飲門市，被 91APP 以 $32M 全現金併購，間接證明了這個模型的可規模化。

**Ragic（no-code 資料庫 SMB）**：人頭計價（per-user），分免費 / Lite / Profession / Enterprise 階層，年付給折扣。它對「訂閱厭惡」的破解是**「資料永遠是你的」**——關了訂閱，使用者可以匯出全部資料。這個承諾大幅降低了「萬一被綁架怎麼辦」的恐懼，**讓「訂閱」感覺更像「租用」而不是「上鉤」**。

**漸強實驗室（LINE OA MarTech SaaS）**：B2B 為主，**年約結構 + 套餐方案 + 高用量另計**。客戶簽年約一次付清、季結對帳，符合台灣中大型品牌的採購流程；中小客戶則用月費套餐 + 加購模式。**關鍵是把「訊息推播成本」拆成 pass-through**——使用者用得多自己付得多，不會吃廠商 margin。

從這三個 case 可以歸納出三種 indie 都用得上的招式：

- **物理感**：把訂閱包成「機器 + 服務」、「設備 + 軟體」、「課程 + 工具」這類有實體錨點的東西
- **資料主權**：明確承諾「關了訂閱資料還是你的、隨時可導出」，並把這個寫在主推文案
- **Pass-through 變動成本**：把外部成本（API 用量、訊息推播、AI 推論）拆成使用者自己付的部分，不要全包進固定月費裡

## Indie 的台灣定價 playbook

把上述觀察落到 indie 的實際定價上，可以分成三條路徑：

### B2C / 個人專業工具

- **預設結構**：Free Trial / 入門 → 月費 → 年費（年付折扣 17–25%）→ Lifetime Deal（限時、限量）
- **價格錨點**：月費抓在 NT$120–490 區間（差不多一份午餐到一頓晚餐），年費抓 NT$1,200–4,990
- **AppSumo[^appsumo] 借鏡**：lifetime deal 對台灣消費者特別有效——一次性買斷符合心理結構，**對 indie 來說則是 cash flow 加速器**。Pieter Levels[^levels] 旗下產品早年靠 AppSumo lifetime deal 啟動的策略在台灣同樣 work
- **退訂友善**：在使用者首頁就放「取消訂閱」按鈕。聽起來反直覺，但台灣使用者在「**找不到取消鈕**」時的退訂率比「**看到取消鈕但沒按**」高得多。把信任度做出來，反而續訂

### B2B SMB

- **預設結構**：年費為主、月費為輔、人頭或門市計價
- **價格錨點**：年費 NT$12,000–60,000 / 公司是一個甜蜜點——對中小企業老闆是「不需要董事會」的決策金額（個人裁量約是月薪的 1/3 以下）
- **採購折扣**：「**現在簽兩年送三個月**」對台灣 SMB 比「打 9 折」有感（因為「送的東西」感覺像額外得到，「打折的東西」感覺像本來就該便宜）
- **發票 + 統編一次到位**：B2B 客戶第一次付款就會問「能不能開三聯式」「我有公司統編」——綠界 / ezPay 自動補就好，不要讓客戶為了帳走流程

### 垂直 niche（餐飲、診所、製造、金融）

- **預設結構**：Setup Fee（一次性導入費 NT$10,000–50,000）+ 年費 + 加值模組單獨計
- **設備 / 安裝錨點**：如果有實體設備（讀卡機、POS、Sensor），讓設備變成銷售故事的核心，月費跟在後面
- **客製化收費**：明確列出「這條 service 多少錢」，客製不要免費送——SMB 老闆對「免費客製」不會感激，反而會無止境提需求
- **政府補助綁定**：上架雲市集、把 75% 補助寫在價格頁，讓使用者看到「政府幫你出 75%」是最強的轉換 hook

## 統一發票與訂閱的衝突

最後一個容易被忽略的細節：**統一發票把訂閱的會計搞複雜**。

- **月付**：每月要不要開發票？對 indie 而言，依靠金流平台（綠界、ezPay）自動代開是唯一合理路徑——自己每月手動開 1,000 個客戶的發票，會吃掉一個 FTE 的時間
- **年付**：年初一次開，但客戶常會要求「分 12 期入帳」（會計處理上比較好看）——這跟你的營收認列也直接相關
- **AI usage-based**[^usage]：儲值制是繞過發票困境的招式——客戶一次儲值 NT$3,000，系統開一張發票；之後扣到 0 為止都不再開發票。AI 推論成本透過儲值的方式 pass-through

> 行政成本是台灣本地 SaaS 的隱性成本。**定價結構選錯，每個月會多耗 20-40 小時人工**——等於你把一個工程師當會計用。

## 把定價當成產品決策

把定價放在最後一篇談，不是因為它最不重要，是因為**它的答案要等其他四篇都決定後才能定**：

- 你的合規結構（個人戶 vs 公司、要不要開發票）→ 影響可不可以收年費 / 月費
- 你的整合層（用 LINE OA 訊息 / Fugle API）→ 影響你的 pass-through 成本結構
- 你的獲客通路（雲市集 / FB 社團）→ 影響你的價格錨點要對標什麼
- 你的客群（B2C / SMB / 垂直 niche）→ 影響上面三條路徑的權重

把這五件事一起做完，才算把「給台灣市場的 indie SaaS」這個題目從想像轉成可執行的產品。Vol.5 的題目選擇 + Vol.2 的金流路徑 + 本期五篇的執行細節，疊起來就是一個 indie 開發者在 2026 進場做台灣本地 SaaS 的完整工具箱。

剩下的事情，就是 ship 出去。

[^retention]: retention（留存率）指特定期間內持續付費或活躍的使用者比例，是訂閱制 SaaS 衡量產品黏著度與長期營收健康度的關鍵指標，常見計算方式包含月留存（MRR retention）與年留存。

[^ragic]: Ragic 為台灣團隊開發、目前客戶涵蓋全球的 no-code 雲端資料庫 SaaS，主打讓非工程師用試算表介面建立企業內部表單與工作流系統，採人頭計價並承諾使用者可隨時匯出全部資料。

[^appsumo]: AppSumo 為美國以「軟體 lifetime deal（一次性買斷終身使用）」為主軸的限時特賣平台，新創 SaaS 常透過 AppSumo 上架換取啟動現金流與初期口碑種子用戶。

[^levels]: Pieter Levels 為知名荷蘭籍獨立開發者，以 Nomad List、Remote OK 等以一人之力營運的 SaaS 聞名，是 bootstrap 與 indie hacker 文化的代表性人物。

[^usage]: usage-based pricing（用量計價）為依使用者實際消耗的 API 呼叫次數、token 數或交易量收費的訂閱模型，2023 年後因 LLM 推論成本可變，廣為 AI SaaS 採用，與傳統固定月費形成混合結構。
