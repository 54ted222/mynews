這一集我們談很多人最常問的：Netlify 到底只適合前端，還是能真的支撐個人 SaaS 的後端需求。答案是，它不是萬能後端，但它把最常見的一批需求做成了「夠快上手、夠容易維護」的組合，對小團隊非常實用。

先看 Functions。你可以把它想成和網站一起部署的 API 層，前端和後端入口放在同一個 repo、同一條部署管線。這會直接減少你在工具和權限設定上的摩擦。接著是 Background Functions，當你的任務不需要即時回傳，像寄信、同步資料、批次處理，就可以丟去背景跑，前端先拿到受理回應。再加上 Scheduled Functions，你就有類似 cron 的排程能力，可以做每日彙整、週報、清理任務，不用再養一台專門跑排程的機器。

另外兩個常被低估的是 Forms 和 Blobs。Forms 讓你用很低成本收 waitlist、詢問單、bug 回報。Blobs 則適合一些讀多寫少的資料情境，比如設定快取、簡單狀態資料。它不是要替代你的主交易資料庫，而是幫你把「不值得立刻重型化」的需求先收住。

Edge Functions 則適合放在請求入口，做地區分流、個人化或重導。但這類能力我建議晚一點再放大，因為它會提高架構複雜度。對個人 SaaS 最好的策略通常是先讓交付穩、再讓功能快、最後才是局部極致優化。

一句話總結：你不需要第一天就把架構做到完美，而是要選一條讓你六個月都能持續出貨的路。Netlify 的內建能力真正的價值，在於它替你省下那層「本來要自己搭、自己維護、自己踩坑」的基建時間。
