把一個聊天框接上 LLM API，不叫做一個 Agent 服務。2026 年業界一句反覆被引用的話是：**在正式環境裡，Agent 不是一段提示詞，而是一套分散式系統，只是剛好由 LLM 擔任規劃者與執行者**。一筆使用者請求，背後可能觸發規劃、工具選擇、執行、驗證、再規劃——Agent 的 LLM 呼叫次數通常是純聊天機器人的 3–10 倍，一個不設限的軟體工程任務光 API 費用就能燒掉 5–8 美元。把這套東西做成「可以對外提供的獨立服務」，要補的遠不只是模型本身。

這一期，我們把獨立 Agent 服務拆成可以分層交付的六個子系統。最上層是**推理核心**：Agent 迴圈與規劃模式，2026 年從 ReAct 一路收斂到「單一 orchestrator 開分身、分身只回傳壓縮摘要」的架構——Anthropic、OpenAI、Cognition、Microsoft、LangChain 全部收斂到同一個模式。往下是**記憶與情境**：一份對企業部署的分析指出，近 65% 的 Agent 失敗來自多步推理時的情境漂移與記憶遺失，而非模型本身不會做。再往下是**工具層**，由 Model Context Protocol 收斂成事實標準——截至 2026 年 3 月，MCP 每月 SDK 下載破 9,700 萬次，被每一家主要 AI 廠商支援。

底層則是讓服務「撐得住」的兩塊：**運行時與持久執行**（Temporal 在 2026 年 2 月以 50 億美元估值募得 3 億美元 D 輪，OpenAI、Replit、Lovable 都把 Agent 建在上面），以及橫跨全棧的**安全護欄**與**可觀測性、評測、成本**。Token 價格會跌，但可靠性工程不會過期——這六層，就是我們這一期要逐一拆開看的東西。
