上一集我們談了威脅模型，搞清楚了到底在防什麼——防一個會動手的系統被外部文字劫持。那今天這一集，我們來聊架構：知道了在防什麼之後，這個護欄到底該擺在系統的哪一層？

先打破一個最常見的誤會。很多人一聽到護欄，腦中浮現的畫面就是：在模型輸出後面接一個過濾器，把不該說的話擋掉。問題是，這個位置只能看到「模型說了什麼」，它看不到使用者餵了什麼、看不到模型打算呼叫哪個工具、也看不到工具回傳了什麼。而 agent 出事的環節，恰恰就散落在這整條鏈上。所以只有一個過濾器，是遠遠不夠的。

真正能扛事的做法叫縱深防禦。它的核心主張很簡單：沒有任何單一一種護欄技術能涵蓋全部威脅，所以你要在資料流的不同階段佈下好幾道彼此獨立的關卡。二零二六年業界大致把它細化成四個攔截點，剛好對上 agent 運作的四個位置。

我一個一個講。第一個是輸入端，就是在模型看到任何東西之前，先做注入偵測、輸入分類，還有很重要的一件事——把從外部檢索進來的內容標上來源，讓系統知道哪些字串是「不可信的資料」、而不是「指令」。第二個是推理端，也就是工具呼叫之前，模型決定要動手的那一刻，先過工具白名單、檢查參數合不合法、有沒有越界。這部分的細節是下一集行動層的主題，這裡只先標明：推理端一定要有一道工具控制。第三個是輸出端，模型生成之後、回給使用者之前，驗一下裡面有沒有夾帶個資、機密、或是幻覺。

第四個攔截點最容易被忽略，但它其實是間接注入的主防線，就是工具執行之後。工具回傳了結果、但還沒進到模型腦袋之前，先掃描裡面有沒有藏注入指令、把個資塗掉、看看有沒有外洩的跡象。為什麼一定要擋在「進腦袋之前」？因為模型在架構上就是分不清指令跟資料，兩個都在同一個脈絡裡。這也是為什麼像 Snowflake 這類廠商會特別把這層做在編排層，攔截工具回應，在它碰到底層模型之前先用一個專門訓練過的模型判斷有沒有惡意指令。

我這邊用講的描述一下這四個攔截點怎麼排。你可以想像一條流水線：使用者輸入先進到第一道輸入護欄，過了才到模型推理；模型要呼叫工具，先過第二道工具呼叫前的檢查；工具跑完，回傳的東西要過第四道執行後的掃描，乾淨了才回到模型;最後模型的輸出再過第三道輸出護欄，才回給使用者。任何一道攔下來，就走阻擋或改寫的分支。四個點各守一段，少一個就漏一類攻擊。

接下來講第一個工程上的反作用力：延遲。你想想，如果這四層每一層都丟一個大模型去判斷，互動式的 agent 會慢到不能用。所以重點不只是鋪幾層，而是每一層內部還要分工，業界叫它成本梯度，由便宜往貴升級。最便宜的是正則和啟發式的快篩，用模式比對抓常見的注入片語，延遲在一毫秒以下，缺點是脆，換個寫法就漏。中間一檔是微調過的小型分類器，大概 BERT 那個等級，能抓改寫過的攻擊，延遲十到三十毫秒，但碰到全新型態的攻擊就會明顯退化。最貴的是用一個獨立的大模型當判官，它最準、能讀語意、能抓沒見過的攻擊，但延遲動輒八百毫秒以上，成本是分類器的幾十倍。所以合理的架構是：便宜的快篩全部都跑，只有快篩已經覺得可疑、或是即將執行高風險動作的少數案例，才升級到大模型判官。把貴的留給值得重檢的，不然延遲跟帳單會把整個產品壓垮。

再來有一個比較細、但很實際的問題：串流。互動式 agent 為了體驗，會一個字一個字地把回應吐給你，可是傳統的輸出護欄要等整段都生成完才能驗——這代表內容早就串出去了,護欄才跑完,根本來不及擋。解法是把驗證改成非同步的，跟生成並行：一邊串流給使用者，護欄在背景同時跑，一旦事後發現違規就把已經顯示的內容收回或修正。但這不是免費的，它換來的是「使用者可能短暫看到違規內容、再被收回」的視窗期。所以非同步只適合用在那種「先看到沒有立即危害、事後修正可以接受」的輸出，像是文字回覆。至於轉帳、刪檔、寄信這種一發出去就不可逆的動作，絕對不能非同步，必須是同步、阻塞式的核可，寧可讓使用者多等一下。簡單講，每一層要同步還是非同步，就看那一層失敗了能不能收回來。

最後一個架構決策，是當護欄自己壞掉、超時、或被打爆的時候，要放行還是攔下？放行叫 fail-open，保護的是可用性，但護欄失效時就等於沒護欄；攔下叫 fail-closed，保護的是安全，但合法動作會被卡住。沒有標準答案，要看那一層擋的是什麼。低風險的輸出過濾可以 fail-open，漏一句總比卡住整個產品好；高風險的執行層就該 fail-closed，再配一條明確的人工 override 路徑。不過要小心，fail-closed 本身會開新的攻擊面：既然壞了就攔，攻擊者就可以故意餵會打爆護欄的輸入,把它變成一種阻斷服務的武器,單一份被投毒的文件就可能塞爆共用的護欄基礎設施,把同一台機器上其他 agent 全餓死。

所以這一集的重點是：分層架構真正的紀律，不是鋪愈多層愈安全，而是每一層都要想清楚——它擋的是什麼、付出多少延遲、壞掉的時候往哪邊倒。而且最重要的一條：把那些不可逆的動作，留在最嚴格、同步、而且 fail-closed 的那一層。那這個「最危險的動作層」到底怎麼守，就是下一集行動層護欄要好好展開的東西了。
