前面四集我們把「怎麼裝護欄」講完了——威脅模型、分層架構、行動層控制、還有工具鏈。今天這最後一集，要回答唯一一個真正難的問題：你怎麼知道它有效？

先說一個讓人有點不安的事實。有一份二零二六年的調查訪問了九百多位高管跟技術人員，結果是八成八的組織，在過去一年確認或懷疑發生過 AI agent 的安全事件；可是同一批人裡，只有百分之十四點四的 agent，是在通過完整的安全跟 IT 核可之後才上線的。換句話說，絕大多數組織的 agent 已經在動手了，但沒有人能拿出證據說「它不會做壞事」。所以裝了護欄，真的不等於安全。今天我講三條取得證據的路線——紅隊、評測、治理，而且這三條都有各自的天花板，我會一併講清楚。

第一條線是紅隊。邏輯很簡單：在真的攻擊者之前，自己先用攻擊者的手法，把自己的 agent 打一遍。對 agent 來說，這比傳統的 LLM 紅隊難一個量級，因為你不只要測它會不會說錯話，還要測它會不會在多步推理裡被間接注入帶偏、會不會誤用工具、會不會洩漏憑證、會不會被污染的記憶帶著走。好消息是現在有現成的清單可以用——OWASP 這個資安社群在二零二五年底發布了專門針對「自主、會用工具的 agent」的十大風險清單，把目標劫持、工具濫用、委派信任、agent 之間的通訊攻擊、記憶污染這些抽象的焦慮，翻成了可以逐項驗證的測試項目。工具面，有 promptfoo，它內建了對應這份清單的攻擊插件，能自動跑、還能產出對齊標準的合規報告，補充一個產業訊號，OpenAI 在二零二六年三月把它收購了，但專案維持開源授權。另外還有 DeepTeam 也是類似的開源紅隊框架。二零二六年最明顯的轉變是「自主紅隊」——你給一個扮演攻擊者的模型一句自然語言目標，由它自己去選攻擊、組合變形、打靶、產出發現，攻擊面比人手寫的固定腳本廣得多。

但這裡要踩個煞車。紅隊有一個根本限制：覆蓋率永遠不完整。它只能證明「我測過的攻擊被擋住了」，它沒辦法證明「不存在我沒想到的攻擊」。學界還觀察到自動紅隊有個毛病，就是策略容易缺乏多樣性，會過度集中在某一類指標、忽略其他威脅。所以當紅隊跑出一片全綠的時候，正確的解讀是「這批已知攻擊目前擋得住」，而不是「我們安全了」。

第二條線是評測。一次性的紅隊有個問題——它會過期。你今天測過的版本，明天改一個提示、加一個工具、換一個底層模型，攻擊面就變了。所以驗證必須變成持續的流程：把護欄跟紅隊的用例納入持續整合的測試管線，每次改動就回歸跑一遍，把安全測試當成單元測試在管。比較成熟的團隊會抓一個節奏：每次改提示就跑、每晚對正式環境跑、每季再擴大範圍跑。

不過評測有個比覆蓋率更隱蔽的陷阱，就是指標會被人鑽空子。這是「古德哈特定律」的標準演出——當一個度量變成目標，它就不再是個好度量。一旦「紅隊通過率」或「判官評分」變成團隊的 KPI，被優化的就會是那個數字本身，而不是它原本想代表的「安全」。學界已經有具體觀察：agent 會學著去對那些「看得到的成功」最佳化，而系統性地忽略那些沒被測量的維度，像是合規、安全、程序完整性。而且用來當判官的那個模型，本身也是攻擊面，它會誤判、可以被刻意對抗、輸出還可能因為講得落落長就被高估。對策不是放棄評測，而是承認指標只是代理、不是真相——定期更換保留的攻擊集、引入對中間過程的稽核、不要把單一一個通過率當成放行的唯一憑證。

第三條線是治理跟合規，就是把上面這些變成組織拿得出手、外部查得了的東西。核心是稽核軌跡，一份防竄改、按時間順序記錄 agent 每一個動作的紀錄——觸發了什麼、每一次工具呼叫、存取了哪些資料、決策理由、人類授權的脈絡、時間戳、執行結果。它跟一般 log 的差別在於目的：稽核軌跡是為了問責跟法律上站得住腳，不只是除錯。法規這條，歐盟的 AI 法案是當前最硬的壓力，它已經生效，多數規則二零二六年八月起全面適用，違反禁止性行為最高可以罰到全球年營業額的百分之七。治理框架面，美國 NIST 的 AI 風險管理框架、還有可以被認證的 ISO 42001 管理系統標準，都提供了可操作的對映。

但這條線最大的誤區，是把合規當成安全。通過稽核、留好軌跡、對齊框架，證明的是「出事的時候你說得清楚、可以被問責、罰則可控」——這是可問責性，不是攻不破。法案要你保存紀錄、做風險管理，它不會替你擋住一個新型的間接注入。反過來也成立：一個技術上很硬的 agent，要是沒有稽核軌跡，出事時你連發生了什麼都查不出來。合規跟安全是兩個正交的維度，缺一不可，但誰都頂替不了誰。

那我們把這整輯收一下。回到最開頭那句話——當 AI 從會說話變成會動手，一句被注入的指令就能變成一筆真實操作。這一期五集，我們從威脅模型出發，搞懂了爆炸半徑為什麼會炸開；然後用分層架構決定護欄擺哪裡；再進到最危險的行動層，管住動手那一刻；接著挑對工具鏈把它做出來；最後用紅隊、評測、治理去證明它真的有用。而這最後一集最誠實的答案是：你永遠沒辦法證明 agent 絕對安全，你只能持續地縮小未知。紅隊去找新攻擊、評測讓驗證持續又自動、治理讓組織可問責，三者互相補對方的盲點，但三者也都有天花板。所以重點是——把「紅隊覆蓋率不完整、評測會被鑽空子、合規不等於安全」這三句話，當成要持續運作的提醒，而不是可以打個勾就結案的項目。能做到這一點，才是 agent 時代護欄工程真正成熟的樣子。這一期就到這裡，謝謝你聽完。
