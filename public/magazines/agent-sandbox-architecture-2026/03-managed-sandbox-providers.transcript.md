上一集我們把四種隔離原語攤開來比,知道了強隔離不一定慢、但每種都有取捨。那這一集我們聊一個更實際的問題:如果你不想自己架 Firecracker、自己調 gVisor,市面上有沒有現成的可以買?有,而且到二零二六年已經是個成熟市場了,主要有五家——E2B、Modal、Daytona、Vercel 還有 Cloudflare。這一集我們就來看,什麼情況該選誰。

但我得先講一個很重要的前提:這五家不是同一種東西的五個牌子。它們其實是三種不同的生意,撞在同一張比較表上。

第一種生意,是「純執行原語」,代表是 E2B 跟 Daytona。E2B 建在 Firecracker microVM 上,每個沙箱一顆專屬核心,是這幾家裡隔離最硬的,定位就是給 AI 跑未審查程式碼的臨時環境。Daytona 呢,它二零二五年從開發環境工具轉型過來,主打全市場最快的冷啟動,但代價是它預設用的是 Docker 容器,也就是共享主機核心,隔離比較弱;你要 microVM 等級的安全,得自己選 Kata。

第二種生意,是「運算平台順手賣沙箱」,代表是 Modal。它是個 Python 原生的 serverless 平台,沙箱用 gVisor,但它真正的賣點不是隔離,而是 GPU 全家桶,還有能擴展到五萬個以上並發 session 的規模。適合你想把 agent 推理跟重運算放在同一個平台。

第三種生意,是「既有平台的延伸」,代表是 Vercel 跟 Cloudflare。Vercel Sandbox 底層也是 Firecracker,但它的價值主張就一句話:你本來就在 Vercel 上嘛。Cloudflare Sandboxes 是二零二六年四月才正式商用的,架構很有意思,是兩層:外層用 Workers 的 V8 isolate 做路由跟輕量邏輯,內層才是容器化的沙箱跑真正的程式碼,賣點是跟它的邊緣網路綁在一起。

所以選型的第一刀,往往不是比數字,而是先決定你要哪一種生意。

決定完生意型態,我們再來看最容易被誤導的地方:定價。這裡有個坑,你一定要知道。如果你把單價放在一起看,會覺得 E2B 比 Vercel 便宜一半多。但這是個陷阱,因為兩家算的「小時」根本不是同一段時間。

我這樣講你就懂了。E2B 跟 Daytona,算的是「牆鐘時間」,就是沙箱從開到關的整段真實時間,不管 CPU 在不在忙,都照算。但 Vercel 跟 Cloudflare 算的是「active CPU」,也就是只算你的程式碼「真的在燒 CPU」的那幾秒,等 I/O 的時間不算。這個差別有多大?你想想 agent 在幹嘛——它大量的時間是在等,等 LLM 回應、等 API、等資料庫。這段等待的時間,Vercel 不收你 CPU 費,但牆鐘計費的 E2B 照收。所以對一個「忙百分之五、等百分之九十五」的互動式 agent,active CPU 的模型很可能便宜得多。但反過來,如果你是那種 CPU 從頭忙到尾的批次轉檔、編譯、跑測試,牆鐘計費反而帳單更平、更好估。所以單價不能直接相減,你得先搞清楚是哪一種計費,再用你自己真實的「忙跟等」的比例去算。

再來聊一個對 agent 特別重要的功能:快照。對 agent 來說,快照不是錦上添花,而是省錢又省延遲的核心機制。你想嘛,裝完依賴、跑完設定,存一個快照,下次直接還原,就跳過了整段冷啟動跟安裝。但各家「快照」的含義差很多。最完整的是 E2B,它的暫停跟恢復,捕捉的不只是檔案系統,還包含「完整的記憶體狀態」——執行中的程式、載入的變數,通通在內。恢復大概一秒,就回到你暫停當下那一刻,最長可以存三十天。這種「連記憶體一起凍結」的能力,是讓 agent 跨 session 接續、還有做分叉的基礎。相比之下,Vercel 的快照就偏向「環境快照」,它存的是檔案系統跟裝好的套件,語意上更接近「裝好的鏡像」,而不是「凍結的執行狀態」。

那把這些收斂成可以執行的決策,大概是這樣。如果你跑的是 agent 隨手生成、完全沒審查的程式碼,安全是第一順位,那就選 E2B,Firecracker 給你的逃逸門檻最高;如果你硬要用 Daytona,務必自己去開 Kata,別用它的預設。如果你的產品面向終端使用者,每一次點擊都要在一百毫秒內起一個沙箱,那選 Daytona,它約九十毫秒的冷啟動是最硬的賣點,前提是你能接受用隔離換速度。如果你的工作吃 GPU、或要同時開到上萬個 session,選 Modal。如果你的團隊已經重押在 Vercel 或 Cloudflare 生態裡,那就直接用同名的 Sandbox,省下來的整合跟維運成本,通常比單價那點差更值錢。

最後給三個提醒,免得你被比較文帶偏。第一,那些冷啟動數字,幾乎全部來自第三方 benchmark 或某家自己的部落格,而且高度依賴 region、鏡像大小、有沒有預熱,所以把它當量級看,別當合約上的承諾。第二,單價不能直接相減,記得先確認是牆鐘還是 active CPU。第三,這個市場二零二六上半年還在劇烈變動,光 Cloudflare 兩個月內就棄用了通訊協定跟一部分 SDK,所以你導入前,務必回官方頁面核一次最新數字。好,知道了去哪裡租,下一集我們把鏡頭拉回最貼身的地方——你自己的筆電。
