前三集我們聊的,基本上都是「別人的機器替你跑 agent」。但二零二六年,絕大多數開發者每天打交道的,其實是跑在自己筆電上、直接讀寫你工作目錄的本機 coding agent。這一集我們就來看這個——你的 Claude Code、OpenAI 的 Codex,還有一個叫 nono 的工具,它們怎麼把跑在你機器上的 agent 給關住。

這時候情況不一樣了。你沒有 microVM、沒有獨立的虛擬機邊界可以用。唯一能把 agent 關住的,是作業系統自己內建的安全模組。在 macOS 上,叫 Seatbelt;在 Linux 上,是 bubblewrap,搭配兩個機制,一個叫 Landlock、一個叫 seccomp。你不用記這些名字,只要知道一件事:這些東西的價值,在於「強制點」不在模型身上,而在核心裡。Claude Code 的文件講得很好:權限規則是在指令跑之前、看字串判斷的;但沙箱邊界,是作業系統施加在「執行中行程」上的。所以不管模型決定跑什麼、就算被允許的指令做了名字以外的壞事,這道邊界依然成立。它不靠 LLM 自律,它靠核心。

那本機沙箱到底要擋什麼?危險來自一個很簡單的事實:agent 是用「你的權限」在跑。它能讀你的 SSH 金鑰、讀你的雲端憑證,能把東西寫進你的設定檔,能在安裝套件的時候,在那個安裝腳本裡偷偷連到任意網址。所以本機沙箱要同時管兩件事:檔案系統,還有網路。少了網路隔離,被攻陷的 agent 能把你的金鑰外洩;少了檔案隔離,agent 能改系統設定,繞回去重新拿到網路。兩道都要到位。

我們先看 Claude Code,我把它的特色叫做「可以退回的沙箱」。它的檔案系統預設是「寬讀、嚴寫」——寫入只限你當前的工作目錄,但讀取是整台電腦。注意喔,文件自己明白提醒,這個預設「仍然允許讀取你的雲端憑證跟 SSH 目錄」,你要擋,得自己手動列出來。所以憑證外洩的風險,在預設下是開著的。網路這邊,它走的是一個「沙箱外的代理」,由這個代理依網域名稱強制白名單。但這道門有個官方自己點名的弱點:這個內建代理不終止、也不檢查 TLS 流量,它只看 client 自己回報的網域名稱。這意味著什麼?意味著沙箱裡的程式,可以用一種叫 domain fronting 的手法,表面上連一個被允許的大網域,實際上連到別的地方。所以你只要把像 github.com 這種大網域加進白名單,就等於開了一條潛在的外洩路徑。

而 Claude Code 最有特色、也最有爭議的,是它的逃生口。文件寫得很清楚:有些指令根本沒法在沙箱裡跑,與其讓任務失敗,當指令因為沙箱限制失敗時,Claude 會分析原因,然後可能用一個叫 dangerouslyDisableSandbox 的參數去重試——重試的指令在沙箱外跑,所以會回到一般的權限流程,需要你按同意。這是個很務實的設計,它讓沙箱可以預設關得更緊,因為總有後路。但安全意涵也很實在:這道沙箱不是一面不可逾越的牆,而是一道「預設關住、卡住就申請開門」的閘門。你想拿掉這條後路也行,管理端可以設定成嚴格模式,讓那個逃生參數被完全忽略。

再來看 Codex,它的差異化定位是「預設就把沙箱打開」。它在 git 工作區下,預設就是只能寫工作區、預設沒有網路;你要開網路,得自己明確去設。這跟 Claude Code「預設不開、要你主動啟用」剛好相反,所以 Codex 常被稱為唯一一個預設啟用沙箱的主流 agent。它在 Linux 上的實作是多層疊加的:檔案系統預設把整個根目錄唯讀掛載,連 symlink 逃逸、還有「先建路徑再寫」這類手法都堵死;行程隔離上,它隔離 user 跟 PID 的 namespace,還施加一個機制,叫 PR_SET_NO_NEW_PRIVS,意思是禁止這個行程透過任何手段重新拿回特權,等於把提權的後門焊死。

第三個工具最特別,叫 nono。它跟前兩個性質完全不同——它不是一個 coding agent,而是一個你拿來「包住」任何 agent 的外殼。它的賣點,剛好對著 Claude Code 的逃生口:不可逆。技術底層大家都一樣,Linux 用 Landlock、macOS 用 Seatbelt,但 nono 把這套原語「一次性、不可撤回」的特性,當成核心賣點。它的官方說法是,不同於那種攔截並過濾操作的沙箱,nono 是打造一個「未授權操作在結構上根本不可能」的環境。你用的時候,是明確地授予路徑、網路跟指令的存取,然後一旦套用,限制就不可逆了。文件寫得很白:「沒有放寬的 API,沒有逃生口,連 nono 自己都拿不掉。」

所以你把 nono 跟 Claude Code 並排,一條很清楚的軸線就出來了:可逆,還是不可逆。Claude Code 那個逃生口是刻意保留的後路,對開發體驗很友善,但這代表只要使用者按下同意,被攻陷的 agent 就能在沙箱外跑指令——換句話說,人類那一句「好」,變成了邊界的一部分。而 nono 把這條後路徹底封死,就算 agent 或攻擊者拿到完整控制權,也沒有任何路徑能放寬限制,因為「放寬」這個能力在核心層根本不存在。代價是彈性:用 nono,你得在啟動前就把整段工作會用到的路徑跟網路,全部想清楚、一次給足,跑到一半發現少給了,只能重來。

這也是本機沙箱的核心取捨。可逆的沙箱,把「要不要放寬」交給人在當下判斷,順手,但把人為誤判也納進了威脅面;不可逆的沙箱,把判斷提前到啟動時、之後概不通融,安全上更硬,但要求你對 agent 的需求有近乎完整的先驗知識。沒有哪個對所有情境都對。所以這一集的重點是,選型的時候你一定要認清:你那道「沙箱」,到底是一面拆不掉的牆,還是一道隨時能被一個「Yes」打開的閘門?好,從威脅、到原語、到市場、到你的筆電,我們都走過一遍了。最後一集,我們把所有東西疊起來,談談怎麼真的把沙箱經營成一層能上線的基礎設施。
